**Three lifecycle hook overrides (per D-05, D-06, D-07, D-08):**

1. `_resolve_expressions(self) -> None` -- Override to resolve context variables on scalar config fields ONLY (die_on_error, rows_buffer_size, label, enable_auto_convert_type). Skip Java expression resolution entirely because tMap's expressions reference row data (row1.column, lookup1.column) that don't exist at config resolution time. Do NOT call `super()._resolve_expressions()`. Only use `self.context_manager.resolve_string()` on the scalar fields if `self.context_manager` is not None.

2. `_select_mode(self, input_data) -> ExecutionMode` -- Always return `ExecutionMode.BATCH`. tMap handles its own chunking internally.

3. `_update_stats_from_result(self, result: dict) -> None` -- Iterate ALL keys in result except "stats". For each value that is a non-empty DataFrame, add `len(value)` to total_rows. Check if the output name corresponds to a reject output (is_reject or inner_join_reject in config["outputs"]) and track reject_rows separately. Update `self.stats["NB_LINE"]`, `self.stats["NB_LINE_OK"]`, `self.stats["NB_LINE_REJECT"]`. This implements MAP-07.

**_validate_config(self) -> None (per D-09):**
Validate:
- `config["inputs"]` must exist and be a dict
- `config["inputs"]["main"]` must exist with a "name" key
- `config["inputs"]["lookups"]` must be a list (can be empty)
- Each lookup must have "name", "join_keys" (list), "join_mode"
- Each join_key must have "lookup_column" and "expression"
- `config["outputs"]` must be a list with at least one entry
- Each output must have "name" and "columns" (list)
- Raise `ConfigurationError(f"[{self.id}] ...")` with descriptive messages

**_process(self, input_data=None) -> dict (per D-09, D-21, D-22):**
This is the core processing method. ALL processing state must be local variables -- never store on self. The method receives a dict of DataFrames from OutputRouter and returns a dict of DataFrames.

Implementation flow:
1. **Parse input:** Accept `input_data` as either `Dict[str, DataFrame]` or single `DataFrame`. Extract main_df by looking up the main input name. Extract lookup DataFrames by name. If main_df is None or empty, return empty outputs.

2. **Apply main input filter:** If `config["inputs"]["main"]["activate_filter"]` and filter expression exists, evaluate via Java bridge preprocessing to get boolean mask, filter main_df.

3. **Process lookups sequentially** (preserve chained lookup support -- Lookup2 can reference Lookup1 columns):
   For each lookup in `config["inputs"]["lookups"]`:
   a. Get lookup_df from inputs dict
   b. Apply lookup filter if `activate_filter` is True (via Java bridge preprocessing)
   c. **Smart join routing** -- classify join type per D-15:
      - Call `_classify_join_type(join_keys)` which checks each key's expression:
        - If `_strip_java_marker(expr)` matches `_SIMPLE_COLUMN_RE` -> equality key
        - If expression contains only `context.` or `globalMap.get` references -> context_only key
        - Otherwise -> cross_table key
      - Route to appropriate handler:
        - `_join_equality()`: pandas merge
        - `_join_context_only()`: evaluate context expressions, filter lookup, cross-join
        - `_join_cross_table()`: Java bridge preprocessing for both sides, merge on evaluated columns
   d. **Null key pre-filter (MAP-03):** Before any merge, split both main and lookup on null keys. `null_mask = df[key_columns].isna().any(axis=1)`. Non-null rows go through merge. Main rows with null keys are tracked for inner join reject if applicable.
   e. **Matching mode handling (MAP-01, D-11, D-14):**
      - `UNIQUE_MATCH`: `lookup_df.drop_duplicates(subset=key_columns, keep='last')` -- Talend's HashMap.put overwrites, last row wins. Research confirmed this. Current code was correct.
      - `FIRST_MATCH`: `lookup_df.drop_duplicates(subset=key_columns, keep='first')`
      - `LAST_MATCH`: `lookup_df.drop_duplicates(subset=key_columns, keep='last')`
      - `ALL_MATCHES`: no dedup (cartesian product possible -- apply size guard)
   f. **Column prefixing:** Before merge, prefix ALL lookup columns with `{lookup_name}.{column_name}` to prevent collisions (per research Topic 5).
   g. **Perform pandas merge:** `pd.merge(joined_df, lookup_df, left_on=left_keys, right_on=right_keys, how='left', indicator=True)` 
   h. **Inner join reject tracking (MAP-02, D-12):** If `lookup["join_mode"] == "INNER_JOIN"`, rows with `_merge == "left_only"` are inner join rejects. Track these in a `inner_join_reject_dfs` dict keyed by lookup name. For LEFT_OUTER_JOIN, no reject tracking needed. Drop the `_merge` column after tracking.
   i. **RELOAD_AT_EACH_ROW handling (MAP-08, D-27):** If `lookup["lookup_mode"] == "RELOAD_AT_EACH_ROW"`, do NOT use bulk merge. Instead, iterate each main row: set globalMap variables from the row, re-evaluate the lookup filter expression against the full lookup DataFrame, perform single-row join. This is O(n*m) but matches Talend semantics for parameterized lookups. Log a warning for large datasets (main > 10K rows AND lookup > 10K rows).

4. **Compute which main rows failed ALL inner joins combined:** A main row that failed ANY inner join on ANY lookup is flagged as rejected. Combine null-key rejects + merge-unmatched rejects.

5. **Evaluate variables** (per research Topic 3): Variables are evaluated AFTER all lookups, BEFORE output expressions. Variables can reference earlier variables (dependency chains). Use Java bridge preprocessing to evaluate variable expressions sequentially on the joined_df. Store evaluated variables as columns in joined_df with `Var.{variable_name}` prefix.

6. **Generate and compile output script** (per D-04, D-19, D-28):
   - Build a Java/Groovy script string for the compiled execution:
     - Define output struct classes with column names
     - For each non-reject, non-inner_join_reject output: evaluate column expressions
     - Use `IntStream.range(0, rowCount).forEach()` (NOT `.parallel()` -- fixes BUG-MAP-003)
     - Each iteration creates a local `Var` HashMap for variable storage
     - Wrap expression evaluation in try-catch if ANY output has `catch_output_reject=True` (MAP-05)
   - Call `self.java_bridge.compile_tmap_script(component_id=self.id, ...)` to compile once
   - Call `self.java_bridge.execute_compiled_tmap_chunked(component_id=self.id, df=joined_df, chunk_size=chunk_size, schema=schema_dict)` to execute in chunks
   - The result is `Dict[output_name, DataFrame]`

7. **Apply output filters:** For each output with `activate_filter=True`, evaluate the filter expression. Rows passing the filter go to the output. Rows failing the filter go to outputs marked with `is_reject=True` (if any such output exists).

8. **Route catch output rejects (MAP-05, D-24):** If compiled script execution caught expression errors, route error rows to outputs with `catch_output_reject=True`. Add `errorMessage` column to the error DataFrame.

9. **Route inner join rejects:** Route inner_join_reject_dfs to outputs with `inner_join_reject=True`.

10. **Auto type conversion (MAP-06, D-25):** If `config.get("enable_auto_convert_type", False)` is True, before merge in step 3g, cast join key columns to compatible types. Handle common mismatches: str->numeric (pd.to_numeric with errors='coerce'), numeric->str (astype(str)), int->float, float->int.

11. **Build result dict:** Return `{output_name: DataFrame for each output}`. Include all named outputs, reject outputs, and inner join reject outputs.

**When Java bridge is not available:**
If `self.java_bridge` is None, the component should still work for simple column-reference expressions (no Java evaluation needed). For expressions that require Java evaluation, log a warning and skip that expression (set the output column to None). This allows testing join logic without a running JVM.
