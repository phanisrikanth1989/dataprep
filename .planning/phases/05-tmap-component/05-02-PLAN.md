**Fixtures:**

`_DEFAULT_CONFIG` -- A valid tMap config with 1 main (row1) + 1 lookup (row2) + 1 output (out1). Use simple equality join on a column called "key". Output has one column "val" mapped from "row1.val". No filters, no variables, UNIQUE_MATCH, LOAD_ONCE, LEFT_OUTER_JOIN.

`_make_component(config=None, global_map=None, context_manager=None, java_bridge=None)` -- Factory creating Map instance with fresh GlobalMap and ContextManager by default. Uses component_id="tMap_1".

`_make_main_df(rows=None)` -- Creates a main input DataFrame with columns: id (int), key (str), val (int). Default 3 rows: (1, "A", 100), (2, "B", 200), (3, "C", 300).

`_make_lookup_df(rows=None)` -- Creates a lookup DataFrame with columns: key (str), label (str). Default 3 rows: ("A", "Alpha"), ("B", "Beta"), ("D", "Delta").

`_make_input_dict(main_df=None, lookup_df=None, main_name="row1", lookup_name="row2")` -- Wraps DataFrames in a dict matching OutputRouter format: `{main_name: main_df, lookup_name: lookup_df}`.

**Important testing notes:**
- For tests that need Java expression evaluation (filters, complex expressions), the component should fall back to a no-bridge path for simple column references. Tests that only use simple column refs (like `row1.key`) can work without a Java bridge mock by having the component extract the column reference directly from the expression string.
- Tests requiring actual Java bridge evaluation should be marked `@pytest.mark.java` and skipped if the bridge is unavailable.
- Most tests should work without a Java bridge by using simple column reference expressions that the component can resolve internally (strip `{{java}}` prefix, match `table.column` pattern, extract column value directly from DataFrame).
- Use `copy.deepcopy(_DEFAULT_CONFIG)` for each test to prevent cross-test contamination.
