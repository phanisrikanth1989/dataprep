## Cross-Cutting Constraints (apply to every plan)

1. **Doc-only phase (D-E1).** No `src/` changes anywhere. If reconciliation surfaces a real production bug, capture it inline in the audit doc with `[NEW IN 15.1]` severity tag AND in the plan SUMMARY as a Phase-16-or-later follow-up. No defensive shims, no source patches.
2. **ASCII-only (D-C4).** No emoji, no smart quotes, no en/em dashes; use `--`. Verify via `grep -nP "[^\x00-\x7F]" <file>` returning zero lines. Mirrors Phase 15 D-C1 + project memory `feedback_ascii_logging`.
3. **Section-preserving edits (D-A3).** A section is only edited when its underlying content actually changed. Section 3 ("Talend Feature Baseline") is the high-stability section per 15.1-RESEARCH.md Section B -- default to untouched unless Talaxie semantics actually shifted (D-B2). Sections 5, 6, 9, 10 are the high-update zones.
4. **Resolution-marker tag format (D-C1).** `~~original text~~ [RESOLVED in Phase N, commit <short_sha> (BUG-ID)]`. Net-new findings get `[NEW IN 15.1] severity: text`. Both coexist in the same P0/P1/P2/P3 block.
5. **Verify-before-claim (D-C3).** Every cited path / class / function / Talaxie reference is grep-confirmed BEFORE the commit lands. Carries Phase 15 D-E2 + project memory `feedback_verify_audit_claims`. SHAs MUST be checked via `git show <sha> --stat | grep <expected file>` before being written into a strike-through.
6. **Atomic commits (D-C6).** One commit per audit doc reconciled. One commit per cross-cutting doc regenerated. Closeout splits into ~5 commits by logical change (REQUIREMENTS, ROADMAP, STATE, VERIFICATION, PHASE-SUMMARY). Mirrors Phase 14 D-F2 / Phase 15 D-E1.
7. **No new top-level docs (D-A2).** No `FINDINGS.md`, no `ROLLUP.md`, no per-category index files. Every finding lands INLINE in an existing audit doc's existing P0/P1/P2/P3 section.
8. **No non-shipped audit doc edits (D-A5).** Plans must NOT touch the 20 non-shipped audit docs. Their broken refs are enumerated as known limitations in 15.1-VERIFICATION.md (closeout) for future roadmap reference.
9. **No CLAUDE.md edits (D-E4).** Mirrors Phase 15 D-B4.
10. **No top-level docs/ edits (D-E5).** The 4 canonical docs + root README from Phase 15 (`docs/ARCHITECTURE.md`, `docs/COMPONENT_REFERENCE.md`, `docs/CONTRIBUTING.md`, `docs/DEPLOYMENT.md`, `README.md`) stay locked.
