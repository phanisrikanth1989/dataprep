<objective>
Fix three correlated defects in `Executor._execute_iterate_body` (CR-02, CR-03, CR-04):

- **CR-02:** The loop uses `enumerate(iter_component.iteration_iter, start=1)` directly, bypassing the documented `has_next_iteration()` / `get_next_iteration_context()` public API. `current_iteration_index` permanently stays 0. Fix: drive the loop via `while iter_component.has_next_iteration()` + `iter_component.get_next_iteration_context()`.

- **CR-03:** The `except ComponentExecutionError` arm after `_execute_subjob_plan(body_plan)` is dead code because `_execute_component` catches all exceptions and returns `"error"`. `on_iteration_error` (Hook 8) therefore never fires. Fix: remove Hook 8 (`on_iteration_error`) from `BaseIterateComponent` and remove the unreachable `except ComponentExecutionError` arm from the iterate loop. Document the errors-as-statuses contract explicitly in the module docstring.

- **CR-04:** `_any_body_die_on_error` reads `self.execution_stats[body_id]` which is never cleared between iterations. Stale `"error"` status from iteration N can cause spurious die_on_error termination in iteration N+1. Fix: track per-iteration failed body IDs in a local set `iter_local_failed_bodies`; check `die_on_error` only against that set.

Purpose: Restores the documented API contract, eliminates dead code, and prevents stale-stats early termination.
Output: Corrected executor, updated base class docstring, new tests proving all three fixes.
</objective>

```python
def has_next_iteration(self) -> bool:
    """Check if there are more iteration items to process.
    For bounded iterators (total_iterations >= 0), uses self.current_iteration_index.
    Returns True if more items may exist.
    """
    if self.total_iterations == -1:
        return True
    return self.current_iteration_index < self.total_iterations

def get_next_iteration_context(self) -> Dict[str, Any]:
    """Get next iteration item and advance index. Sets globalMap vars.
    Calls set_iteration_globalmap(item), advances current_iteration_index,
    writes f"{self.id}_CURRENT_ITERATION" to globalMap (ITER-11 / D-F7).
    Returns dict with 'item' and 'index' keys, or empty dict if exhausted.
    """
    if not self.has_next_iteration():
        return {}
    try:
        item = next(self.iteration_iter)
    except StopIteration:
        return {}
    self.set_iteration_globalmap(item)
    self.current_iteration_index += 1
    if self.global_map:
        self.global_map.put(f"{self.id}_CURRENT_ITERATION", self.current_iteration_index)
    return {"item": item, "index": self.current_iteration_index}
```

```python
def _any_body_die_on_error(
    self, body_plan: SubjobPlan, iter_failed_bodies: set[str] | None = None
) -> bool:
    """Check if any body component that failed THIS iteration has die_on_error=True.

    Uses iter_failed_bodies (IDs that changed to 'error' this iteration) to avoid
    reading stale execution_stats from prior iterations (CR-04).

    Args:
        body_plan: The body subgraph plan.
        iter_failed_bodies: Set of body component IDs that failed in the current
            iteration. If None, falls back to reading execution_stats directly
            (legacy behavior -- only safe if called for the first iteration).

    Returns:
        True if any of the iteration-local failed components has die_on_error=True.
    """
    check_ids = iter_failed_bodies if iter_failed_bodies is not None else set(body_plan.component_ids)
    for body_id in check_ids:
        comp = self.components.get(body_id)
        if comp is None:
            continue
        if getattr(comp, "die_on_error", False):
            return True
    return False
```

Note: The `iter_failed_bodies` approach checks `die_on_error` directly on the component rather than re-reading `execution_stats`, which is cleaner and avoids the stale-stats race entirely.
