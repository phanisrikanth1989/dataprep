**Part B: Full rewrite of join.py**

Replace the entire contents of `src/v1/engine/components/transform/join.py` with a from-scratch rewrite. The new component must:

**Module docstring** (per Rule 12):
```
Config keys consumed (4 total):
  use_inner_join   (bool, default False)    -- True for inner join, False for left outer
  join_key         (list[dict])             -- [{input_column, lookup_column}, ...]
  use_lookup_cols  (bool, default False)    -- include lookup columns in output
  lookup_cols      (list[dict])             -- [{output_column, lookup_column}, ...]
```

**Imports:** logging, typing (Any, Optional), pandas, BaseComponent, REGISTRY, ConfigurationError, DataValidationError.

**Registration:** `@REGISTRY.register("Join", "tJoin")` (per D-14).

**Class: Join(BaseComponent)**

`_validate_config(self) -> None` (per D-02):
- Require `join_key` as a non-empty list.
- Each element must be a dict with `input_column` (str) and `lookup_column` (str) keys.
- Raise ConfigurationError with `[{self.id}]` prefix on any failure.

`_resolve_inputs(self, input_data) -> tuple[pd.DataFrame, pd.DataFrame]`:
- Private helper. `input_data` is a dict from OutputRouter (flow_name -> DataFrame).
- If dict has `main` and `lookup` keys, use directly.
- Otherwise, use `self.inputs` list (set by engine): first = main, second = lookup.
- Raise ConfigurationError if exactly 2 inputs not available or either is None.
- Return (main_df, lookup_df) tuple.

`_process(self, input_data=None) -> dict` (core logic):

1. Call `_resolve_inputs(input_data)` to get main_df, lookup_df.
2. Read config values: `use_inner_join` (default False), `join_key`, `use_lookup_cols` (default False), `lookup_cols` (default []).
3. Extract main_key_cols and lookup_key_cols from join_key list.
4. **First-match dedup** (per Pitfall 7): `lookup_df = lookup_df.drop_duplicates(subset=lookup_key_cols, keep='first')`.
5. **Null key sentinel** (per D-03, JOIN-08): Define module-level constant `_NULL_SENTINEL = "__DATAPREP_NULL_SENTINEL__"`. Before merge, on COPIES of the DataFrames, replace NaN in key columns with `_NULL_SENTINEL` using `.fillna()`. After merge, any row where any key column equals `_NULL_SENTINEL` is treated as non-match.
6. **Merge**: Use `pd.merge()` with `left_on=main_key_cols`, `right_on=lookup_key_cols`, `how='left'` (always left, even for inner join -- use indicator to filter), `indicator=True`, `suffixes=('', '_lookup')`.
7. **Classify rows from indicator**: `matched = merged['_merge'] == 'both'`, `unmatched = merged['_merge'] == 'left_only'`.
8. **Sentinel filtering**: For rows where matched is True, if any key column equals `_NULL_SENTINEL`, move those rows to unmatched (null keys must not match).
9. **Build main output**: If `use_inner_join`: `main_out = merged[matched]`. Else (left outer): `main_out = merged` (all rows).
10. **Build reject output**: `reject_rows = merged[unmatched]`. Select only original main columns for reject base.
11. **INCLUDE_LOOKUP toggle** (per D-05, JOIN-04): If `use_lookup_cols` is True AND `lookup_cols` is non-empty, keep only: all main columns + specified lookup output columns (mapped from `lookup_cols` list: each dict has `output_column` and `lookup_column` -- the `output_column` is the column name in the output, `lookup_column` is the source column from lookup). If `use_lookup_cols` is False, keep only main-side columns in output (no lookup columns).
12. **Drop merge artifacts**: Remove `_merge` indicator column and any sentinel remnants (restore NaN). Drop duplicate lookup key columns that differ from main key columns.
13. **Reject schema** (per D-08, JOIN-03): If `self.reject_schema` exists and is non-empty, reindex reject to match reject schema columns. Add `errorCode='JOIN_REJECT'` and `errorMessage='No matching lookup row'` columns.
14. **ERROR_MESSAGE globalMap** (per D-06, JOIN-05): If any error occurs during processing, set `{self.id}_ERROR_MESSAGE` in globalMap. On success with rejects, also set it to indicate rejected rows exist.
15. **Stats**: Call `self._update_stats(len(main_df), len(main_out), len(reject_rows))`.
16. Return `{"main": main_out, "reject": reject_rows if not reject_rows.empty else None}`.

**Error handling**: Wrap merge logic in try/except. On exception: check `self.config.get('die_on_error', True)` -- note: die_on_error is not a converter param for tJoin, so it comes from BaseComponent lifecycle (default True). Set `{self.id}_ERROR_MESSAGE` in globalMap. Re-raise as ComponentExecutionError if die_on_error, else return empty main and full main as reject.

**Do NOT:**
- Override execute() (Rule 4).
- Store state on self (Rule 10).
- Use print() (Rule 8).
- Read UPPERCASE config keys -- use lowercase converter keys directly (per D-02).
- Implement case-insensitive join by default -- Talend tJoin is case-sensitive (Java string equality). The converter does NOT output a case_sensitive flag (phantom param removed). Case-sensitive matching is the default. The temp column approach from D-04 should be available as a method but only activated if a `case_sensitive` config key is explicitly set to False (future-proofing).
