- No config keys to validate (schema drives behavior). This is a pass-through -- `pass` is acceptable but log a debug message for traceability.

`_process(self, input_data=None) -> dict`:
- If `input_data` is None or empty DataFrame: return `{"main": input_data, "reject": None}`.
- Read `self.output_schema` (set by engine). If not set or empty, return input as-is (passthrough).
- Extract column names: `schema_cols = [col["name"] for col in self.output_schema]`.
- Select only columns present in both schema and input: `available = [c for c in schema_cols if c in input_data.columns]`.
- If `available` differs from `schema_cols`, log warning listing missing columns.
- Return `{"main": input_data[available].copy(), "reject": None}`.
- Call `self._update_stats(len(input_data), len(input_data), 0)` (all rows pass, just columns change).

Do NOT: include `mode`, `columns`, `keep_row_order` config keys. Do NOT store NB_COLUMNS_IN/NB_COLUMNS_OUT in globalMap (not Talend behavior). Do NOT override execute(). Do NOT use print().

- `input_data` will be a dict of DataFrames from OutputRouter (multi-input component).

- `combined = pd.concat(dfs, ignore_index=True, sort=False)`.
- `total_input = sum(len(df) for df in dfs)`.
- Call `self._update_stats(total_input, len(combined), 0)`.
- Return `{"main": combined, "reject": None}`.

Do NOT: include `mode`, `remove_duplicates`, `sort_output`, `merge_columns`, `merge_how`, `keep`, `sort_columns` config keys. Do NOT define `__init__` with `self.input_data_map`. Do NOT override execute(). Do NOT define `_process_batch`, `_process_streaming`, or `add_input` methods. Do NOT use print(). Do NOT store `{id}_INPUT_COUNT`, `{id}_MODE`, `{id}_INPUT_ROWS`, `{id}_OUTPUT_ROWS` in globalMap (not Talend behavior).
