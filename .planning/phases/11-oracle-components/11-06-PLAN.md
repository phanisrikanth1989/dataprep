for "no engine yet") with a conditional Wallet/OCI thick-mode `needs_review` entry that

configured for SID/SERVICE_NAME/RAC should convert clean (no false-positive review
flags). Jobs configured for OCI/Wallet should still surface a review flag because the
engine raises ConfigurationError until thick-mode is set.
