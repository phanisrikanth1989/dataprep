<objective>
Implement memory-bounded chunked cross-product (D-05) and rewrite
`_join_cross_table` to use it. Fix the issue 2c crash. Add the back-compat
config knob for chunk sizes.

Purpose:
- Issue 2c (Job_filter_join: empty join_keys + filter as match) currently
  crashes with `list index out of range` -- a bug in the dispatch when
  `join_keys` is empty AND `activate_filter` is True. The dispatch (now
  locality-aware after plan 02) needs an explicit empty-keys + filter branch.
- Today's `_join_cross_table` materializes the full main x lookup cross-
  product as a single DataFrame -- for a 1M x 100K join, that's 100B rows in
  memory. D-05 mandates streaming/chunked iteration: per-chunk cross-product
  -> bridge eval -> filter -> append -> concat at the end.

Output:
- New `_chunked_cross_product(main_df, lookup_df, match_expr, main_name, lookup_name, joined_lookup_names) -> pd.DataFrame` helper.
- New `_compute_cross_chunk_size(lookup_rows) -> int` static helper.
- `_join_cross_table` rewritten to delegate to `_chunked_cross_product`.
- Empty-keys + filter branch added to dispatch in `_process` (fixes issue 2c crash).
- `output_chunk_size` config key with `rows_buffer_size` legacy alias.
  Default drops from 2,000,000 to 50,000.
- `cross_join_chunk_size` config key (auto-tuned via the formula, override-able).
</objective>

    2. **Add `_chunked_cross_product`** per PATTERNS.md Analog B "New method shape":
       ```python
       def _chunked_cross_product(
           self,
           main_df: pd.DataFrame,
           lookup_df: pd.DataFrame,
           match_expr: Optional[str],
           main_name: str,
           lookup_name: str,
           joined_lookup_names: list[str],
       ) -> pd.DataFrame:
           if lookup_df.empty:
               # Cross-product with empty lookup is empty.
               return main_df.iloc[0:0]
           # Allow override via config; auto-tune as fallback.
           chunk_size = int(
               self.config.get("cross_join_chunk_size",
                   self._compute_cross_chunk_size(len(lookup_df)))
           )
           result_chunks: list[pd.DataFrame] = []
           for start in range(0, len(main_df), chunk_size):
               chunk = main_df.iloc[start:start + chunk_size]
               chunk_cross = pd.merge(chunk, lookup_df, how="cross")
               if match_expr is None:
                   result_chunks.append(chunk_cross)
                   continue
               eval_results = self._bridge_eval(
                   chunk_cross,
                   {"__match__": self._strip_java_marker(match_expr)},
                   joined_lookup_names + [lookup_name],   # D-04: full list
               )
               if "__match__" in eval_results:
                   mask = pd.Series(eval_results["__match__"]).fillna(False).astype(bool).values
                   result_chunks.append(chunk_cross[mask])
           if not result_chunks:
               return main_df.iloc[0:0]
           return pd.concat(result_chunks, ignore_index=True)
       ```
       Note the `joined_lookup_names + [lookup_name]` arg -- the cross-product
       contains the current lookup, so the bridge needs all of them.
