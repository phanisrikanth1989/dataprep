# Phase 05.3 Plan 05: Pure-Python Eval Path for tMap No-Marker Dispatch (D-02) Summary

Pure-Python eval path for tMap (D-02): when no `{{java}}` markers are present anywhere in the tMap config, expressions for outputs, variables, and filters are evaluated via Python `eval()` without any JVM interaction.

## What Was Built

### New Module-Level Classes

**`_VarBag`** - Live attribute-access wrapper around a shared dict. Unlike `types.SimpleNamespace` (which copies kwargs at construction time), `_VarBag` reads from the live dict on every `__getattr__`. This enables sequential Var chaining: `Var.v2 = Var.v1 + "_USD"` works because `v1` is visible in the same dict when `v2` is being evaluated.

**`_NullNamespace`** - Read-only namespace where any attribute access returns `None`. Used for lookup tables that had no matching rows (empty lookup or unmatched left-outer rows). Implements Talend's left-outer null semantics: `row2.label` returns `None` instead of `AttributeError` when the lookup had no data.

### New Methods

- **`_build_row_dicts_for_py`** - Splits a joined DataFrame row into per-table dicts. For main table: takes all columns not prefixed by a lookup name. For each lookup: strips the `"row2."` prefix and extracts those values.
- **`_build_namespace`** - Builds the sandboxed `eval()` namespace per row. Uses `SimpleNamespace` for tables with data, `_NullNamespace` for empty tables (empty lookup / unmatched left-outer). Uses `_VarBag` for `Var`. Uses `_build_safe_builtins()` for security.
- **`_eval_expr`** - Safe `eval()` call with `die_on_error` gate: raises `ComponentExecutionError` when `die_on_error=True`, logs warning and returns `None` when `False`.
- **`_apply_filter_py`** - Row-level Python filter eval. Detects single-table vs joined-df context (based on whether prefixed columns exist for any lookup name) to correctly build row dicts for RELOAD_AT_EACH_ROW (unprefixed columns) vs standard joined context.
- **`_evaluate_variables_py`** - Sequential variable eval returning `dict[str, list]` (one list per variable, one value per row). Builds a fresh `_VarBag` per row so each row gets independent sequential mutation.
- **`_eval_output_row`** - Evaluates all column expressions for a single row using the prebuilt namespace.
- **`_evaluate_outputs_py`** - Main output builder for the no-marker path. Handles pre-computed var columns, per-row namespace building, output filter application, and reject routing.

### Wiring Changes

**`_process` dispatch**: No-marker path now calls `_evaluate_variables_py` (returns var columns dict) and `_evaluate_outputs_py` instead of the bridge-based `_evaluate_outputs`.

**`_apply_filter`**: Added `has_marker` check at the top. No-marker expressions route to `_apply_filter_py`; marker expressions continue through the bridge path.

### Deleted

**`_evaluate_outputs_simple`** - Replaced entirely by `_evaluate_outputs_py`. The old method used a per-column bridge fallback that returned `None` when bridge was unavailable (Issue 1 root cause). The new method uses pure Python eval with no bridge dependency.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] `_apply_filter_per_row` stripped marker before calling `_apply_filter`**
- **Found during:** Bridge smoke test after GREEN commit
- **Issue:** `_apply_filter_per_row` called `self._strip_java_marker(filter_expr)` before passing to `_apply_filter`. Before this plan, `_apply_filter` always stripped and routed to bridge, making double-strip harmless. After adding the `has_marker` dispatch in `_apply_filter`, the pre-stripped expression always had `has_marker=False`, routing all RELOAD_AT_EACH_ROW filter expressions to `_apply_filter_py` instead of the bridge.
- **Fix:** Detect marker presence in `_apply_filter_per_row` before substituting row refs, then re-attach `{{java}}` prefix after substitution so `_apply_filter` sees the original marker and dispatches correctly.
- **Files modified:** `src/v1/engine/components/transform/map.py`
- **Commit:** `cc45e19`

**2. [Rule 1 - Bug] Empty lookup tables produced `AttributeError` not `None`**
- **Found during:** GREEN phase test run (16 failures)
- **Issue:** `_build_row_dicts_for_py` built `row_dicts["row2"] = {}` for empty/skipped lookups. `_build_namespace` created `SimpleNamespace()` from that empty dict. Any expression accessing `row2.label` then raised `AttributeError`. For left-outer join with `die_on_error=True`, this crashed instead of returning `None`.
- **Fix:** Added `_NullNamespace` class (any-attr returns `None`). `_build_namespace` uses `_NullNamespace` when `row_data` is empty.
- **Files modified:** `src/v1/engine/components/transform/map.py`
- **Commit:** `b84010a`

**3. [Rule 1 - Bug] `_apply_filter_py` single-table context detection for RELOAD_AT_EACH_ROW**
- **Found during:** GREEN phase test run (7 failures in TestReloadAtEachRow)
- **Issue:** `_apply_filter_py` was called from `_apply_filter_per_row` with an unfiltered `lookup_df` whose columns are unprefixed (`key, label, active`). But `_build_row_dicts_for_py` was checking for `row2.`-prefixed columns, finding none, and assigning everything to main. So `row_dicts["row2"] = {}` and `row2.label` failed.
- **Fix:** Added context detection in `_apply_filter_py`: check if any column in df starts with `"{ln}."` for any lookup name. If so: joined-df context, use `_build_row_dicts_for_py`. If not: single-table context, all columns belong to `table_name`.
- **Files modified:** `src/v1/engine/components/transform/map.py`
- **Commit:** `b84010a`

**4. [Rule 1 - Bug] `_evaluate_outputs_py` test setting `config["die_on_error"]` not `comp.die_on_error`**
- **Found during:** GREEN phase test run (TestPlan1406bUnitGapClosure failure)
- **Issue:** Test called `_evaluate_outputs_py` directly (bypassing `execute()`). `_eval_expr` reads `self.die_on_error` which is initialized to `True` and only updated from config during `execute()`. Test set `comp.config["die_on_error"] = False` but not `comp.die_on_error = False`.
- **Fix:** Updated test to also set `comp.die_on_error = False`.
- **Files modified:** `tests/v1/engine/components/transform/test_map.py`
- **Commit:** `b84010a`
