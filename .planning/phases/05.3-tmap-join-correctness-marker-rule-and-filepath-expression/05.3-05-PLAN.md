    3. **Add `_VarBag` helper class** (small, near other module-level helpers in map.py):
       ```python
       class _VarBag:
           """Live attribute-access wrapper around a single shared dict.

           Used for the tMap `Var` namespace in the python-eval path.
           Sequential variable evaluations mutate the same backing dict
           via __setattr__, so later variable expressions see earlier
           assignments through __getattr__.

           Why not types.SimpleNamespace? SimpleNamespace.__init__(**d) copies
           keys onto the instance at construction time. Mutating the original
           dict afterward does NOT propagate. That breaks chained Var eval
           (Var.v2 = Var.v1 + "_USD" sees Var.v1 as undefined).
           """
           __slots__ = ("_d",)

           def __init__(self, d: dict | None = None):
               object.__setattr__(self, "_d", d if d is not None else {})

           def __getattr__(self, name: str):
               # Called only when normal attribute lookup fails.
               try:
                   return self._d[name]
               except KeyError:
                   raise AttributeError(name)

           def __setattr__(self, name: str, value) -> None:
               self._d[name] = value
       ```
       Place this near the existing module-level helpers (e.g., next to `_ROW_REF_PATTERN`).

           self,

           ns: dict = {}
           ns.update(_SAFE_NAMESPACE_GLOBALS)
