<objective>
Fix issue 6: tFileOutputDelimited filepath written as literal expression
string (` + output_via_expr + .csv`) instead of evaluating `context.X +
context.Y + context.Z`. Per D-06: converter marks the field, engine evaluates
on demand.

Purpose:
- The converter today emits `filepath` as a plain string from `_get_str`.
  When the Talend job uses an expression like `context.dir + context.name +
  context.ext`, the engine receives the literal string and after context
  resolution writes a file named ` + output_via_expr + .csv`.
- Per D-06: the converter should use `ExpressionConverter.mark_java_expression`
  (which already detects `+`, ternaries, `.method()`, routines) and prefix
  with `{{java}}` only when needed. The engine, after context resolution,
  honors the marker by sending to the bridge for evaluation.

Output:
- Converter `src/converters/talend_to_v1/components/file/file_output_delimited.py:62`
  uses `ExpressionConverter.mark_java_expression` for `filepath` and
  `streamname`.
- Engine `src/v1/engine/components/file/file_output_delimited.py` honors the
  marker via a small `_resolve_java_expr_param(value, name)` helper that
  sends a 1-row placeholder DataFrame to `execute_tmap_preprocessing`.
- This plan is INDEPENDENT of all map.py work -- can run in Wave 1 in parallel
  with 05.3-01.
</objective>

    3. **Add helper `_resolve_java_expr_param`** on the component class. Per
       PATTERNS.md "Pattern to insert" + "No Analog Found -> 1-row placeholder":
       ```python
       def _resolve_java_expr_param(self, value: str, name: str) -> str:
           """If `value` starts with the {{java}} marker, send the stripped
           expression to the bridge for context-free evaluation. Returns the
           evaluated string. If no marker, returns value unchanged.

           Uses a 1-row placeholder DataFrame because the bridge API is
           DataFrame-shaped and the expression has no row context.
           """
           if not isinstance(value, str) or not value.startswith(_JAVA_MARKER):
               return value
           if self.java_bridge is None:
               raise ConfigurationError(
                   self.id,
                   f"Param '{name}' has value {value!r} requiring Java bridge "
                   f"evaluation but bridge is unavailable. Either start the "
                   f"bridge (java_config.enabled=true) or remove the Java "
                   f"expression from the job."
               )
           expr = value[len(_JAVA_MARKER):]
           # Pass at least one dummy column so the Arrow schema is non-empty
           # (zero-column DataFrames don't survive Arrow IPC serialization).
           placeholder = pd.DataFrame([{"__placeholder__": 1}])
           try:
               eval_result = self.java_bridge.execute_tmap_preprocessing(
                   df=placeholder,
                   expressions={"__param__": expr},
                   main_table_name="__none__",
                   lookup_table_names=[],
                   schema={},
               )
           except Exception as exc:
               raise ComponentExecutionError(
                   self.id,
                   f"Failed to evaluate {name} expression {expr!r}: {type(exc).__name__}: {exc}",
                   cause=exc,
               ) from exc
           if "__param__" not in eval_result or not eval_result["__param__"]:
               raise ComponentExecutionError(
                   self.id,
                   f"Bridge returned no result for {name} expression: {expr!r}"
               )
           return str(eval_result["__param__"][0])
       ```
