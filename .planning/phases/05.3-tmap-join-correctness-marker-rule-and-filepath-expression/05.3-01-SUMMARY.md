# Phase 05.3 Plan 01: Universal Java Marker Rule and Dispatch Boundary Summary

Universal `{{java}}` marker scanner + hard-fail-on-missing-bridge for tMap, replacing the
per-column hybrid dispatch with a single marker-presence-anywhere signal per D-01/D-02.

## What Was Built

### Task 1: Add `_has_any_java_marker` and hard-fail check; remove dead constant

**`_has_any_java_marker(self) -> bool`** added to `Map` class (replaces deleted `_has_java_expressions`):
- Scans ALL expression-bearing config fields: `inputs.main.filter`, `inputs.lookups[].filter`,
  `inputs.lookups[].join_keys[].expression`, `variables[].expression`, `outputs[].filter`,
  `outputs[].columns[].expression`
- Returns True if ANY field starts with `_JAVA_MARKER = "{{java}}"`
- No per-column simplicity check -- the marker itself is the signal
- Defensive `isinstance(value, str)` guard before `.startswith`

**Hard-fail added to `_validate_config`:**
```python
if self._has_any_java_marker() and self.java_bridge is None:
    raise ConfigurationError(
        f"[{self.id}] tMap config contains {{java}} expressions but Java bridge is unavailable..."
    )
```
Fires at `_validate_config` time (called from `execute()` before `_process()`).

**Dispatch sites updated:**
- `_process()` line ~434: `_use_compiled = self._has_any_java_marker()` with defense-in-depth assertion
- `_evaluate_outputs()`: same replacement; redundant `has_java_expressions` variable removed

**Dead constant removed:** `_DEFAULT_CHUNK_THRESHOLD = 100000` (line 57) deleted.

**9 new tests added** (`TestHasAnyJavaMarker` in `test_map.py`):
- Tests 1-7: each marker-bearing field independently triggers the scanner
- Test 8: marker-present + bridge-None raises ConfigurationError
- Test 9: marker-present + bridge-attached does not raise

### Task 2: Smoke-verify dispatch against live-bridge tests

- 15 of 16 existing `test_map_bridge.py` tests passed immediately after Task 1
- 1 failing test (`test_has_java_expressions_filter_branch`) was testing the deleted method
  and was updated to `test_has_any_java_marker_replaces_has_java_expressions`
- All 16 live-bridge tests now pass
