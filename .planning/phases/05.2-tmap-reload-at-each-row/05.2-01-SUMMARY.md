## Accomplishments
- Fixed pre-filter guard so RELOAD_AT_EACH_ROW mode receives unfiltered lookup data
- Implemented per-row filter evaluation with main row value substitution via new _apply_filter_per_row method
- Added quote-aware expression substitution (_substitute_row_refs) that skips table.column patterns inside string literals
- Replaced string-cast key comparison with type-aware _values_equal supporting int/float/string-numeric coercion
- Fixed unmatched LEFT_OUTER_JOIN rows to include NaN-filled lookup columns with correct column ordering

## Decisions Made
- Used Python-side expression substitution (no per-row JVM round-trips) per D-10 constraint
- Created _apply_filter_per_row as a new method (not extending _apply_filter) to keep interfaces clean
- Quote-aware regex substitution to prevent replacement inside string literals (addresses Gemini review concern)
- Null-safe evaluation with TypeError/ValueError catch returns empty DataFrame (null never matches per MAP-03)
- Float promotion for type-aware comparison (known precision limitation for >2^53 integers, acceptable for ETL join keys)
