### WR-01: extractTypedValue fallback branch may NPE on toString()

**Files modified:** `src/v1/java_bridge/java/src/main/java/com/citi/gru/etl/JavaBridge.java`
**Commit:** 08fe7f2
**Applied fix:** Added null guard in the `else` fallback branch of `extractTypedValue()`. The raw `vec.getObject(rowIndex).toString()` call is now preceded by a null check: `Object raw = vec.getObject(rowIndex); return raw != null ? raw.toString() : null;`. This prevents NPE if an Arrow vector returns null from `getObject()` for a non-null row.

### WR-02: loadNextBatch() return value unchecked -- empty Arrow streams silently produce zero rows

**Files modified:** `src/v1/java_bridge/java/src/main/java/com/citi/gru/etl/JavaBridge.java`
**Commit:** 849c9cd
**Applied fix:** All 4 `reader.loadNextBatch()` call sites (lines 180, 362, 461, 564) now capture the boolean return value and log a WARNING if no batch was loaded. This makes empty/corrupt Arrow streams visible in logs rather than silently producing zero rows.

### WR-03: executeTMapPreprocessing silently drops expressions that fail to compile

**Files modified:** `src/v1/java_bridge/java/src/main/java/com/citi/gru/etl/JavaBridge.java`
**Commit:** 141fe8d
**Applied fix:** Changed the compile error log level from `logger.severe` to `logger.warning` and updated the message to explicitly state that all rows will return null for the failed expression. This properly communicates the silent-null behavior without implying a system-level failure.

### IN-01: Missing vector type handlers for Float4Vector, SmallIntVector, TinyIntVector

**Files modified:** `src/v1/java_bridge/java/src/main/java/com/citi/gru/etl/JavaBridge.java`
**Commit:** b13731f
**Applied fix:** Added three new type-specific branches to `extractTypedValue()` for `Float4Vector` (returns float), `SmallIntVector` (returns short), and `TinyIntVector` (returns byte). These are inserted after the existing `Float8Vector` handler. This prevents numeric values from being coerced to String via the fallback branch if these Arrow types are ever sent by the Python side.
