    ```python
    """Executor: subjob and component execution with error handling.

    Owns the main execution loop. Uses ExecutionPlan for ordering,
    OutputRouter for data routing, TriggerManager for inter-subjob flow.

    Key design:
    - _execute_subjob() is THE building block for Phase 10 iterate support.
      It will be called in a loop per iteration item.
    - OnSubjobOk triggers fire ONLY after ALL subjob components complete.
    - OnComponentOk triggers fire after each individual component.
    - tDie raises ComponentExecutionError with exit_code to stop the entire job.
    """
    from __future__ import annotations

    import logging
    import time
    from typing import Any

    from .base_component import BaseComponent, ComponentStatus
    from .exceptions import ComponentExecutionError, ConfigurationError
    from .execution_plan import ExecutionPlan
    from .output_router import OutputRouter
    from .trigger_manager import TriggerManager
    from .global_map import GlobalMap

    logger = logging.getLogger(__name__)
    ```

    **Executor class:**

    `__init__(self, components: dict[str, BaseComponent], execution_plan: ExecutionPlan, output_router: OutputRouter, trigger_manager: TriggerManager, global_map: GlobalMap)`:
    - Store references to all dependencies.
    - Initialize tracking sets: `executed_components: set[str]`, `failed_components: set[str]`
    - Initialize `execution_stats: dict[str, Any]`
    - Initialize `_job_terminated: bool = False` (set by tDie)

    `execute_job(self) -> dict[str, Any]`:
    - Main entry point. Called by ETLEngine.execute().
    - Start timer.
    - Execute initial subjobs from `execution_plan.initial_subjobs` in order.
    - For each initial subjob: call `_execute_subjob(subjob_id)`, then `_fire_subjob_triggers(subjob_id, result)`.
    - If `_job_terminated` is True after any subjob, stop immediately.
    - After all subjobs, check for runtime stall: if `len(executed_components) < total_component_count`, raise ConfigurationError with list of unexecuted components and their subjob IDs (D-07).
    - Return execution stats dict with keys: 'status' ('success'|'failed'|'error'), 'execution_time', 'components_executed', 'components_failed', 'component_stats'.

    `_execute_subjob(self, subjob_id: str) -> str`:
    - THE building block for Phase 10.
    - Get `SubjobPlan` from `execution_plan.get_subjob_plan(subjob_id)`.
    - Execute components in topological order (`subjob_plan.component_ids`).
    - For each component:
      - Check `output_router.are_inputs_ready(comp_id)`. If not ready and component has inputs, skip with warning (this shouldn't happen with correct topo sort, but safety net).
      - Call `_execute_component(comp_id)`.
      - If result is 'error':
        - Check component's `die_on_error` config. If True (default): mark remaining components in this subjob as skipped, return 'error'.
        - If die_on_error is False (D-11): mark as failed, continue to next component. Skip any component whose inputs depend on the failed component's outputs.
      - After each component: fire OnComponentOk/OnComponentError triggers via TriggerManager. If triggered components are in a DIFFERENT subjob, queue that subjob for execution.
      - If `_job_terminated`: return 'error' immediately.
    - After all components in subjob complete: clear subjob flows via `output_router.clear_subjob_flows(subjob_plan.component_set)`.
    - Return 'success' if no fatal errors, 'error' otherwise.

    `_execute_component(self, comp_id: str) -> str`:
    - Get input data via `output_router.get_input_data(comp_id)`.
    - Start timer.
    - Call `component.execute(input_data)`.
    - Route outputs via `output_router.route_outputs(comp_id, result)`.
    - Store execution stats.
    - Set component status in trigger_manager: `set_component_status(comp_id, 'success')`.
    - Add to `executed_components`.
    - Return 'success'.
    - On exception:
      - Check for tDie: `if hasattr(e, 'exit_code')` -> set `_job_terminated = True`, re-raise to stop the entire job (D-13).
      - Otherwise: `set_component_status(comp_id, 'error')`, add to `failed_components` and `executed_components`, store error stats, return 'error'.

    `_fire_subjob_triggers(self, subjob_id: str, subjob_result: str) -> None`:
    - After subjob completes, check OnSubjobOk/OnSubjobError triggers.
    - Use `execution_plan.get_all_trigger_edges_from_subjob(subjob_id)`.
    - For OnSubjobOk edges: if subjob_result is 'success', call TriggerManager to check if trigger should fire. If yes, execute the target subjob immediately.
    - For OnSubjobError edges: if subjob_result is 'error', do the same.
    - For RunIf edges: evaluate condition via TriggerManager (D-08). If condition is True, execute target subjob. If False, skip (no error).
    - Handle recursion: fired subjobs may themselves have triggers, so process them transitively.
    - Independent subjobs always continue (D-12): a failure in Subjob A does not prevent Subjob C from running if C has no trigger dependency on A.

    Important implementation notes:
    - Do NOT clear subjob flows if flows cross subjob boundaries (when a flow goes from a component in one subjob to a component in another -- check if all consumers are within the same subjob before clearing).
    - The `die_on_error` check must read from the RESOLVED config (after context resolution), not the raw config.
    - When skipping dependents of a failed component: use the execution plan's flow graph to determine which components depend on the failed one's output.
