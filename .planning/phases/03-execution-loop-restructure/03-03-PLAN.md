def _are_inputs_ready(self, comp_id):
    component = self.components[comp_id]
    if not component.inputs:
        return True
    for input_flow in component.inputs:
        if input_flow not in self.data_flows:
            return False
    return True

def _get_input_data(self, comp_id):
    component = self.components[comp_id]
    if not component.inputs:
        return None
    if len(component.inputs) == 1:
        return self.data_flows.get(component.inputs[0])
    else:
        return {input_flow: self.data_flows.get(input_flow) for input_flow in component.inputs}
```

    """Output routing: manage data flows between components.

    Replaces inline routing in engine.py _execute_component() (lines 542-559)
    and _get_input_data()/_are_inputs_ready() (lines 739-769).
    """
    from __future__ import annotations

    import logging
    from typing import Any, Optional

    import pandas as pd

    logger = logging.getLogger(__name__)
    ```

    **OutputRouter class:**

    `__init__(self, flows_config: list[dict], components_config: list[dict])`:
    - Store flows_config and build pre-computed lookup structures:
      - `_outgoing: dict[str, list[dict]]` -- comp_id -> list of outgoing flow configs
      - `_incoming: dict[str, list[str]]` -- comp_id -> list of incoming flow names
    - Build component outputs lookup: `_component_outputs: dict[str, list[str]]` -- comp_id -> component's declared outputs list
    - Build component inputs lookup: `_component_inputs: dict[str, list[str]]` -- comp_id -> component's declared inputs list
    - Build flow-to-consumer mapping: `_flow_consumers: dict[str, set[str]]` -- flow_name -> set of component IDs that consume this flow. Built by iterating flows_config and collecting the `to` component for each flow name.
    - Initialize `_data_flows: dict[str, Any] = {}` -- the data store

    `route_outputs(self, comp_id: str, result: dict[str, Any]) -> None`:
    - Route component result to named flows based on flow config.
    - For each outgoing flow from comp_id:
      - flow type 'flow' -> store `result.get('main')` if not None
      - flow type 'reject' -> store `result.get('reject')` if not None
      - flow type 'filter' -> store `result.get('main')` if not None
      - flow type 'iterate' -> store `result.get('iterate')` if not None
    - For named outputs: iterate result keys that are not in ('main', 'reject', 'stats'). If the key is in the component's declared outputs, store by key name. Otherwise store as `{comp_id}_{key}`.
    - Skip any value that is None.
    - Log at DEBUG level: "Routed {comp_id} output to flow {flow_name}"

    `get_input_data(self, comp_id: str) -> Optional[Any]`:
    - Get input data from upstream flows.
    - If component has no inputs: return None.
    - If single input: return `_data_flows.get(input_name)` directly (the DataFrame).
    - If multiple inputs: return dict keyed by flow name -> DataFrame.

    `are_inputs_ready(self, comp_id: str) -> bool`:
    - If component has no inputs: return True.
    - Return True if ALL input flow names have data in `_data_flows`.

    `clear_flow(self, flow_name: str) -> None`:
    - Remove a flow's data. Used for iterate cleanup between iterations.
    - `_data_flows.pop(flow_name, None)`

    `clear_subjob_flows(self, subjob_component_ids: set[str], executed_components: set[str] | None = None) -> None`:
    - Clear outgoing flows from components in the given set, BUT preserve flows that have cross-subjob consumers that have NOT yet executed. (Addresses review concern: cross-subjob flow lifecycle).
    - For each comp_id in subjob_component_ids, for each outgoing flow:
      - Get the consumers of this flow from `_flow_consumers`.
      - If ALL consumers are within `subjob_component_ids` (i.e., no cross-subjob consumers), safe to clear.
      - If ANY consumer is outside `subjob_component_ids` AND that consumer is not in `executed_components` (i.e., it hasn't run yet), PRESERVE the flow. Log at DEBUG level: "Preserving flow {flow_name}: cross-subjob consumer {consumer_id} has not yet executed".
      - If ALL consumers (including cross-subjob ones) are in `executed_components`, safe to clear.
    - Per D-16 research: free subjob data after subjob completion when safe.
    - Log at DEBUG level how many flows were cleared vs preserved.

    `get_flow_data(self, flow_name: str) -> Optional[Any]`:
    - Direct access to a specific flow's data. Used for edge cases.

    `has_flow_data(self, flow_name: str) -> bool`:
    - Check if a flow has data available.

    `get_pending_flow_names(self) -> list[str]`:
    - Return list of flow names that currently have data stored. For diagnostic purposes (stall detection).
