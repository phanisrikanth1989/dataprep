## Implementation Details

### Vectorized path (ENG-CR-02, ENG-WR-01)

```python
splits = (
    df[norm_col]
    .astype("string")    # coerces any cell type (list, NaN, int) safely -- no pd.isna crash
    .fillna("")
    .str.split(sep, regex=False)  # literal separator, not regex
)
out = df.explode(norm_col, ignore_index=True)
```

The `.astype("string")` call neutralizes the ENG-WR-01 array-truthiness crash because it converts any cell value (including lists) to its string representation before splitting. `.explode()` preserves dtypes of all non-normalized columns, fixing ENG-CR-02.

### Talend operation ordering (ENG-WR-03)

```python
if discard_trailing:
    splits = splits.apply(self._strip_trailing_empties)
if trim:
    splits = splits.apply(lambda lst: [v.strip() for v in lst])
if dedupe:
    splits = splits.apply(lambda lst: list(dict.fromkeys(lst)))
```

Source: Talaxie tNormalize_main.javajet and help.qlik.com tNormalize documentation.

### Trailing-only empty removal (ENG-CR-03)

`_strip_trailing_empties` mirrors the `lastNoEmptyIndex_` loop in Talaxie source: iterates from the end of the list, stops at first non-empty element. Interior empty strings are preserved.

### Empty-result zero-row behavior (ENG-WR-02)

When `discard_trailing=True` and all split values are trailing empties, `_strip_trailing_empties` returns `[]`. `.explode()` on an empty list produces NaN. The implementation then drops NaN rows via `dropna(subset=[norm_col])`, emitting 0 rows per Talend behavior.

### ConfigurationError contract (ENG-CR-01)

`_validate_config()` now raises `ConfigurationError` and returns `None` on success, matching the BaseComponent abstract contract (Phase 1 D-13).
