  <action>
    Refactor `_fast_path_convert` per WR-04 fix snippet:

    Currently the function iterates per-column, filters per-column with `result = result[good_mask].copy()`, leaving stale indices and possible row_dict gaps for columns checked in later iterations.

    Fix: collect rejected indices first, filter once at end:
    ```python
    def _fast_path_convert(self, df, output_schema):
        result = df.copy()
        bad_indices: set[int] = set()
        reject_rows: list[dict] = []

        for col_def in output_schema:
            col = col_def["name"]
            # Try vectorized path
            try:
                result[col] = self._vectorized_convert(result[col], col_def)
            except (ValueError, TypeError):
                # Per-row fallback for this column
                for idx in result.index:
                    if idx in bad_indices:
                        continue
                    try:
                        result.at[idx, col] = self._convert_value(
                            result.at[idx, col], col_def
                        )
                    except (ValueError, TypeError) as e:
                        bad_indices.add(idx)
                        # Capture row_dict from ORIGINAL df (not result, which may
                        # have partial conversions for other columns)
                        row_dict = df.loc[idx].to_dict()
                        row_dict["errorCode"] = "TYPE_COERCION_FAILED"
                        row_dict["errorMessage"] = f"Column '{col}': {e}"
                        reject_rows.append(row_dict)

        # Filter once + reset
        if bad_indices:
            result = result.drop(index=list(bad_indices)).reset_index(drop=True)

        reject_df = pd.DataFrame(reject_rows) if reject_rows else None
        if reject_df is not None:
            reject_df = reject_df.reset_index(drop=True)

        return result, reject_df
    ```

    Constraints:
    - Preserve existing return tuple shape (likely `(main_df, reject_df)`).
    - Use `errorCode`/`errorMessage` reject columns per Phase 4 D-06/07 + Phase 7.1 D-21 (BaseComponent will rename user columns named `errorMessage` → `errorMessage_user` automatically; this code does not need to handle that).
    - Run TestRejectFlow::test_reject_indices_reset — expect GREEN.
