### WR-15 — INSTR bounds check ignores user start parameter
VERDICT: REAL

Source location: `StringHandling.java` line 592

Current buggy code:
```java
int defaultStart = 1;  // defaultStart is still 1 here
...
if (isVacant(string) || isVacant(search_value) || Math.abs(defaultStart) >= string.length()) {
    return null;  // bounds check uses literal 1, not user's start
}
if (start != null && start != 0) {
    defaultStart = start;  // assignment happens AFTER the guard — too late
}
```

Effect: `INSTR("hello", "x", 100, 1)` silently returns null (happens to work for out-of-bounds)
BUT: `INSTR("hello", "ll", 3, 1)` — the bounds check passes with defaultStart=1 even when user
passes start=3, then substring(defaultStart-1) = substring(0) which starts from index 0, not 3.
The user's start position is not used in the bounds check, causing incorrect results when start > 1.

Fix: Move `if (start != null && start != 0) { defaultStart = start; }` ABOVE the bounds check guard.
Same fix needed for byte[] overload (line 661).

### WR-16 — new Float(double) deprecated constructor
VERDICT: REAL

Source location: `Numeric.java` line 112

Current buggy code:
```java
return new Float(decimal.doubleValue());
```

`new Float(double)` was deprecated since Java 9 and removed in Java 17+. Will throw
`NoSuchMethodError` at runtime on Java 17+ deployments. Use `Float.valueOf((float) ...)`.

Fix: `return Float.valueOf((float) decimal.doubleValue());`
