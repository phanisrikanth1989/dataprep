## Decisions Made

1. **D-21 scope**: The rename of `errorCode`/`errorMessage` applies only to the `main` flow output. Reject flow columns with these names are engine-managed (components add them legitimately in `_process()`). Renaming both flows would break 9 downstream component tests.
2. **Int64Dtype always-nullable**: For nullable int columns, `pd.Int64Dtype()` is always used (not conditional on `isna().any()`). On empty DataFrames, `isna().any()` is False but dtype must still be Int64 for schema correctness.
3. **pandas 3.x compatibility**: `infer_datetime_format` parameter was removed in pandas 3.x. Fallback uses `pd.to_datetime(series, errors="coerce")` directly.
4. **Streaming steps 7b/7c**: For batch mode, steps 7b/7c remain in `execute()` after the batch call. For streaming mode, they run inside `_execute_streaming` per chunk. Batch mode does NOT re-run them after `_execute_streaming` returns (would double-validate after per-chunk validation already ran).

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] D-21 reject flow false-positive: rename scoped to main-only**
- **Found during:** Task 5 (full unit suite run)
- **Issue:** `_rename_reserved_reject_columns` renamed `errorCode`/`errorMessage` in BOTH main and reject flows. Components that legitimately add these to reject output (file_input_delimited, join) had their column names broken, causing 9 previously-passing tests to fail.
- **Fix:** Scoped rename to main flow only. Reject flow is engine-managed; components are expected to set errorCode/errorMessage there.
- **Files modified:** src/v1/engine/base_component.py
- **Commit:** 962508b

**2. [Rule 1 - Bug] pandas 3.x compatibility: infer_datetime_format removed**
- **Found during:** Task 3 (running new tests)
- **Issue:** `pd.to_datetime(series, errors="coerce", infer_datetime_format=True)` raises TypeError in pandas 3.x (parameter removed).
- **Fix:** Changed to `pd.to_datetime(series, errors="coerce")` which uses built-in inference.
- **Files modified:** src/v1/engine/base_component.py
- **Committed in:** 1f7ec81 (inline during GREEN phase)

**3. [Rule 1 - Bug] Int64Dtype conditional logic excluded empty DataFrames**
- **Found during:** Task 3 (TestSchemaHandling::test_empty_df_dtype_preserved)
- **Issue:** `if nullable and df[col_name].isna().any(): astype(pd.Int64Dtype())` — on empty DataFrames, `isna().any()` is False so code fell into `astype("int64")` branch, losing nullable type.
- **Fix:** Changed condition to `if nullable: astype(pd.Int64Dtype())` — always use Int64 for nullable int.
- **Files modified:** src/v1/engine/base_component.py
- **Committed in:** 1f7ec81 (inline during GREEN phase)

**4. [Rule 1 - Bug] StringDtype series `.any()` incompatibility in _apply_treat_empty**
- **Found during:** Task 3 (TestSchemaHandling::test_empty_result_runs_validation)
- **Issue:** `col.apply(lambda v: isinstance(v, str) and v == "").any()` fails on pandas StringDtype columns (TypeError: Cannot perform reduction 'any' with string dtype).
- **Fix:** Added `.astype(bool)` before `.any()`, plus `len(col) == 0` early-exit and try/except wrapper.
- **Files modified:** src/v1/engine/base_component.py
- **Committed in:** 1f7ec81 (inline during GREEN phase)
