### VERDICT: CR-03 — REAL

**Finding:** `DataValidationError` raised inside `_vectorized_convert` (line 661) and `_convert_value` (line 856) cannot be caught by `except (ValueError, TypeError)` at the per-row fallback catch sites (lines 605, 773).

**Confirmation:**
- `exceptions.py`: `DataValidationError(ETLError(Exception))` — NOT a subclass of `ValueError`. Confirmed.
- `file_input_delimited.py` line 661: `raise DataValidationError("Unmapped bool values found")` inside `_vectorized_convert`. The catch at line 605 is `except (ValueError, TypeError)` — mismatch confirmed.
- `file_input_delimited.py` line 856: `raise DataValidationError(f"Empty value for non-nullable column")` inside `_convert_value`. The catch at line 773 is `except (ValueError, TypeError)` — mismatch confirmed.

**Impact:** Any bool column with unmapped values (e.g., "yes"/"no") and any non-nullable empty value crashes the component instead of routing to reject.

**Fix:** Switch `raise DataValidationError(...)` to `raise ValueError(...)` at inner-loop raise sites only.
