## Decisions Made

- **CR-08 exception type preserved as RuntimeException**: Talend upstream wraps `new RuntimeException(e)` identically. Changing to a checked exception would break callers that only handle RuntimeException. Only the message and cause chain improved.
- **CR-07/WR-14/IN-01 not fixed**: Per CLAUDE.md "Talend feature parity is non-negotiable" and MEMORY note `feedback_verify_audit_claims`. All three are byte-identical to upstream per OpenDAS mirror.
- **addDate CR-08 site fixed**: The `addDate(String string, String pattern, ...)` catch block used `string` (not `stringDate`) — the local variable names differ but the fix applies equally.
