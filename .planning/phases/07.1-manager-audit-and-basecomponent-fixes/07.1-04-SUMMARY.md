**One-liner:** Surgical patch of 3 correctness bugs in `file_input_delimited.py`: inner coercion raises switched from `DataValidationError` to `ValueError` (CR-03), fast-path reject collection rewritten to collect-then-drop with pandas 3.0 Arrow compatibility (WR-04), one-time no-schema warning added (WR-06).

## Verdicts (D-13)

All 3 audit findings confirmed REAL:

- **CR-03 REAL:** `DataValidationError(ETLError(Exception))` does not extend `ValueError`. `except (ValueError, TypeError)` at the per-row fallback cannot catch it. Per-row coercion errors crashed the entire component instead of routing to reject.
- **WR-04 REAL:** `_fast_path_convert` used `result = result[good_mask].copy()` mid-loop, leaving stale indices in `result`. The final returned `result` had non-contiguous index (e.g., [2] instead of [0] for 2 rejected rows).
- **WR-06 REAL:** `if not self.output_schema: return df, None` had no warning. Users had no visibility that type coercion was silently skipped.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Removed erroneous self.validate_schema() call in _process**
- **Found during:** Task 3 (reviewing _process structure for die_on_error boundary re-wrap placement)
- **Issue:** `self.validate_schema(main_df, self.output_schema)` was called at line 245 inside `_process`. BaseComponent contract (7.1-01, CONTEXT D-01) states "Subclasses MUST NOT call validate_schema() inside _process" — BaseComponent owns that step via `_apply_output_schema_validation` in step 7c.
- **Fix:** Removed the call and added a comment explaining the correct contract.
- **Files modified:** `src/v1/engine/components/file/file_input_delimited.py`
- **Commit:** 39e5318

**2. [Rule 1 - Bug] pandas 3.0 Arrow string .at assignment incompatibility in WR-04 fix**
- **Found during:** Task 4 (first iteration of _fast_path_convert fix)
- **Issue:** Plan's fix snippet used `result.at[idx, col_name] = self._convert_value(...)` — assigning a non-string (bool, int, float) to an Arrow-backed string column in pandas 3.0 raises `TypeError`. This caused all rows to be rejected.
- **Fix:** Changed approach to accumulate converted values in a `good_converted: dict[int, Any]`, then build a `pd.Series` and assign the full column at once. This avoids the `.at` assignment into typed Arrow columns entirely.
- **Files modified:** `src/v1/engine/components/file/file_input_delimited.py`
- **Commit:** 4c00e45

**3. [Rule 1 - Bug] Test dtype assertion needed pandas 3.0 StringDtype compatibility**
- **Found during:** Task 5 (WR-06 test RED → GREEN)
- **Issue:** Test asserted `dtype == object` but pandas 3.0 Arrow string dtype is `StringDtype(na_value=nan)`, not `object`. The assertion failed even though the column was correctly string-typed.
- **Fix:** Updated assertion to use `pd.api.types.is_string_dtype(dtype)` plus Arrow StringDtype detection.
- **Files modified:** `tests/v1/engine/components/file/test_file_input_delimited.py`
- **Commit:** 1f6ebd8
