## Vendoring Decision

**Option chosen:** Option 2 -- vendor source files under `lib/talend-routines-system/`.

Per 07.1-REVIEWS.md HIGH gap, Talaxie source fetch is the PRIMARY strategy on Mac/Linux. The
previous plan draft listed it as a fallback; the review elevated it to PRIMARY because:
- The Windows local Maven repo at `file:///C:/Softwares/...` is inaccessible on Mac/Linux.
- Source files are human-auditable Java; no decompilation required.
- build-helper-maven-plugin adds the source dir portably via `${project.basedir}`.

The Windows local Maven repo (FALLBACK) is skipped automatically on Mac/Linux because
`[ -d "/C:/Softwares/local-repo" ]` always returns false on Darwin/Linux.

## Package Layout

Files are placed under the **mirrored package layout** (Option b from plan):
```
lib/talend-routines-system/
  routines/
    system/
      RandomUtils.java
      FastDateParser.java
      LocaleProvider.java
      TalendTimestampWithTZ.java
```

This layout makes the package structure explicit and matches Maven source root conventions.
build-helper-maven-plugin adds `lib/talend-routines-system` as an additional source root, so
Maven resolves `package routines.system;` declarations correctly.
