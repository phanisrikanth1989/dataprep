## Implementation Details

### CR-05: list_object -> Python list
Separated the collapsed `if func_name in ("list", "list_object", "union")` branch into three distinct branches. `list_object` now returns `x.dropna().tolist()` (with `ignore_null=True`) or `x.tolist()` (without). `list_delimiter` is NOT applied.

### CR-05-bis: union -> deduplicate + sort + join
`union` now returns `list_delimiter.join(sorted(set(x.dropna().astype(str))))` (with `ignore_null=True`) or `list_delimiter.join(sorted(set(x.astype(str))))`. This correctly distinguishes `union` from `list`.

### WR-09: median warning
Used a closure-based mutable sentinel `_warned = [False]` to emit the warning exactly once per `execute()` call, even when pandas calls the lambda once per group. The sentinel is created fresh each time `_build_agg_func` is called (i.e., once per `execute()`), so re-running the component resets the warning.

### WR-11: groupby sort=False
One-line change in `_grouped_aggregation`: `sort=True` -> `sort=False`. Preserves first-seen group order matching Talend's LinkedHashMap behavior.
