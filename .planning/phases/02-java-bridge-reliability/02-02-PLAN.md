Decompose and rewrite the Java bridge into 4 focused files (per D-07, D-17):

**1. JavaBridge.java (~100-150 lines) -- Gateway entry point:**
- Py4J GatewayServer lifecycle (main method with port argument)
- `setLogLevel(String pythonLevel)` -- maps Python log levels to JUL: "DEBUG"->FINE, "INFO"->INFO, "WARNING"->WARNING, "ERROR"->SEVERE (per D-16)
- Context/globalMap storage as `ConcurrentHashMap<String, Object>`
- Getter/setter methods for context and globalMap: `getContext()`, `getGlobalMap()`, `setContext(key, value)`, `getContext(key)`, `setGlobalMap(key, value)`, `getGlobalMap(key)`
- Delegates data operations to ArrowSerializer
- Delegates expression operations to ExpressionExecutor
- Uses `java.util.logging.Logger` with logger name `"JavaBridge"` (per D-14, D-15)
- Log messages prefixed with `[JavaBridge]` for interleaved log clarity (per D-15)
- `main()` method: parses port from args, starts GatewayServer, logs startup message

**Public methods (dispatching to delegates):**
- `byte[] executeJavaRow(byte[] arrowData, String javaCode, String schemaJson, String inputColumnsJson, String outputColumnsJson)` -> delegates to ExpressionExecutor
- `Object executeOneTimeExpression(String expression)` -> delegates to ExpressionExecutor
- `Map<String, Object> executeBatchOneTimeExpressionsWithGlobalMap(Map<String, String> expressions)` -> delegates to ExpressionExecutor (passes context + globalMap)
- `byte[] executeTMapPreprocessing(byte[] arrowData, Map<String, String> expressions, String schemaJson, String outputColumnsJson)` -> delegates to ExpressionExecutor
- `boolean compileTMapScript(String componentId, String javaScript, String inputColumnsJson, String outputColumnsJson, String lookupColumnsJson)` -> delegates to ExpressionExecutor
- `Map<String, byte[]> executeCompiledTMap(String componentId, byte[] arrowData, String schemaJson, String outputSchemaJson, Map<String, byte[]> lookupData, Map<String, String> lookupSchemasJson)` -> delegates to ExpressionExecutor
- `void loadRoutine(String routineClass)` -> delegates to ExpressionExecutor
- `List<String> validateLibraries(List<String> libraries)` -- validates using `Class.forName()` instead of classpath string matching (per BRDG-04): for each library name, attempt to find a known class (or check if JAR exists on classpath); return list of libraries that could not be validated
- **REMOVE** `executeBatchOneTimeExpressions()` (dead code, per research Dead Code Audit)
- **REMOVE** `executeTMapCompiled()` (superseded by compileTMapScript + executeCompiledTMap pipeline, per research Open Question 2)

**2. ArrowSerializer.java (~250-350 lines) -- Arrow IPC + type mapping:**
- Static utility class, no instance state
- `static Map<String, Object[]> readArrowData(byte[] arrowBytes)` -- reads Arrow IPC stream, returns column name -> Object[] values
- `static byte[] writeArrowData(Map<String, Object[]> data, String schemaJson)` -- writes Arrow IPC stream using schema-driven type mapping
- `static Class<?> schemaTypeToJavaClass(String schemaType)` -- maps schema type string to Java class using switch/case (per RESEARCH Pattern 4):
  - "str"/"string"/"id_string"/"id_character" -> String.class
  - "int"/"integer"/"id_integer" -> Integer.class
  - "long"/"id_long" -> Long.class
  - "float"/"id_float" -> Float.class
  - "double"/"id_double" -> Double.class
  - "bool"/"boolean"/"id_boolean" -> Boolean.class
  - "datetime"/"date"/"id_date" -> java.util.Date.class
  - "decimal"/"bigdecimal"/"id_bigdecimal" -> java.math.BigDecimal.class
  - "id_byte" -> Byte.class
  - "id_short" -> Short.class
  - default -> String.class
- Uses case-insensitive matching (`.toLowerCase()` on input)
- Schema JSON parsing via Gson: `parseSchema(String schemaJson)` returns list of column metadata objects
- Arrow vector creation driven by schema type, not data inference (fixing BRDG-01, BRDG-02)
- Uses `RootAllocator` with 256MB limit (not `Long.MAX_VALUE` -- per security_enforcement note about resource exhaustion)
- All logging via `Logger.getLogger("ArrowSerializer")`

**3. ExpressionExecutor.java (~250-350 lines) -- Groovy compile/execute:**
- `private final GroovyShell shell` initialized once
- Compiled script cache: `ConcurrentHashMap<String, Class<? extends Script>> compiledScriptClasses` -- caches the Script CLASS, not the instance (per BRDG-06 fix). Each execution creates a new instance via `scriptClass.getDeclaredConstructor().newInstance()` -- no synchronization needed.
- `clearCompiledScript(String componentId)` and `clearAllCompiledScripts()` for iterate cleanup
- `executeJavaRow(...)` -- compiles code, iterates rows SEQUENTIALLY (not parallel forEach), creates new Script instance per execution, sets up Binding with row, context, globalMap references
- `executeOneTimeExpression(String expr)` -- parse + run, return result
- `executeBatchOneTimeExpressionsWithGlobalMap(Map<String, String> expressions, Map<String, Object> context, Map<String, Object> globalMap)` -- for each expression, parse + run with context/globalMap in binding, catch errors and return `"{{ERROR}}" + message`
- `compileTMapScript(...)` -- parse Groovy script, cache the Script class in compiledScriptClasses map
- `executeCompiledTMap(...)` -- retrieve cached Script class, create new instance, set binding, execute
- `loadRoutine(String className)` -- attempt `Class.forName(className)` to verify loadability
- Groovy binding setup: `row` (RowWrapper), `context` (Map), `globalMap` (Map), `input_row` (Map alias), `output_row` (Map)
- All logging via `Logger.getLogger("ExpressionExecutor")`
- NO parallel execution (remove `parallelStream`/`forEach` patterns) -- sequential iteration with compile-once optimization provides the real performance win (per research Pitfall 3 / BRDG-06 recommendation)

**4. RowWrapper.java (~100-150 lines) -- Row accessor:**
- Wraps Arrow vectors for per-row access during Groovy expression evaluation
- `RowWrapper(VectorSchemaRoot root)` -- stores reference to root
- `void setRowIndex(int index)` -- sets current row position
- `Object get(String columnName)` -- reads value from the column's vector at current row index
- `void set(String columnName, Object value)` -- writes value to column's vector at current row index
- Type-aware get/set: reads correct Java type from Arrow vector (VarCharVector -> String, IntVector -> Integer, BigIntVector -> Long, Float4Vector -> Float, Float8Vector -> Double, BitVector -> Boolean, TimeStampMilliVector -> Date, DecimalVector -> BigDecimal)
- Implements `propertyMissing(String name)` for Groovy dynamic property access (this is how Groovy scripts access `row.column_name`)
- All logging via `Logger.getLogger("RowWrapper")`

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Denial of Service | ArrowSerializer RootAllocator | mitigate | Use `new RootAllocator(256 * 1024 * 1024)` (256MB limit) instead of `Long.MAX_VALUE` to prevent OOM on malformed input |

| T-02-08 | Information Disclosure | JavaBridge logging | mitigate | Log operation names and row counts, not data values, at INFO level. Data only at FINE/DEBUG. |
</threat_model>
