The module must contain:

1. **SCHEMA_TO_ARROW dict** mapping all known type strings to `pa.DataType`:
   - Converter output types: `"str"` -> `pa.string()`, `"string"` -> `pa.string()`, `"int"` -> `pa.int64()`, `"long"` -> `pa.int64()`, `"float"` -> `pa.float64()`, `"double"` -> `pa.float64()`, `"bool"` -> `pa.bool_()`, `"datetime"` -> `pa.timestamp("ms")`, `"date"` -> `pa.timestamp("ms")`, `"object"` -> `pa.string()`

2. **SCHEMA_TO_JAVA dict** mapping schema type strings to Java class name strings (for the Java side to know what type to expect):
   - `"str"` -> `"String"`, `"int"` -> `"Long"`, `"long"` -> `"Long"`, `"float"` -> `"Double"`, `"double"` -> `"Double"`, `"bool"` -> `"Boolean"`, `"datetime"` -> `"Date"`, `"date"` -> `"Date"`, `"Decimal"` -> `"BigDecimal"`, etc.

3. **`schema_to_arrow_type(col: dict) -> pa.DataType`** function:
   - Reads `col["type"]` (defaults to `"str"` if missing)
   - Handles Decimal/id_BigDecimal specially: reads `col.get("precision", 38)`, clamps precision > 0 (default 38), scale = `min(precision // 2, 18)`, returns `pa.decimal128(precision, scale)`
   - Looks up in SCHEMA_TO_ARROW; if not found, tries case-insensitive lookup with `.lower()`; if still not found, defaults to `pa.string()` with a `logger.warning()`
   - Uses `logging.getLogger(__name__)` for the warning

4. **`build_arrow_schema(columns: list[dict]) -> pa.Schema`** function:
   - Iterates `columns`, calls `schema_to_arrow_type(col)` for each
   - Reads `col.get("nullable", True)` for nullable flag
   - Returns `pa.schema(fields)` where each field is `pa.field(col["name"], arrow_type, nullable=nullable)`

Module-level docstring explaining this is the single source of truth for type mapping per D-05.
Use Python 3.10+ syntax (`dict[str, ...]` not `Dict[str, ...]`).
Use `logging.getLogger(__name__)` at module level.
No print() statements anywhere.

Rewrite `src/v1/java_bridge/bridge.py` from scratch (per D-01). The new bridge must:

**Architecture:**
- Import `build_arrow_schema` from `type_mapping` (same package)
- Use `logging.getLogger(__name__)` -- no print() anywhere (per D-14)
- ASCII-only log messages with `[OK]`, `[ERROR]`, `[WARN]` markers (per D-13)
- Raise `JavaBridgeError` (from `src.v1.engine.exceptions`) on any failure -- never silently fall back (per D-11)
- Use Python 3.10+ type hints (`dict[str, Any]` not `Dict`)

**Constructor `__init__(self)`:**
- `self._gateway: JavaGateway | None = None`
- `self._java_bridge = None` (the Py4J Java object)
- `self._java_process: subprocess.Popen | None = None`
- `self._port: int | None = None`
- `self._context_manager = None` (set externally)
- `self._global_map = None` (set externally)
- `self._base_path = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))`

**Startup `start(self, port: int, log_level: str = "INFO", lib_dir: str | None = None)`:**
- Build classpath from JAR path + optional lib_dir JARs (per BRDG-04)
- Start JVM subprocess with `--add-opens=java.base/java.nio=ALL-UNNAMED`
- Retry connection with exponential backoff: 0.1s, 0.2s, 0.4s, 0.8s, 1.6s (5 attempts max)
- After connection, call `self._java_bridge.setLogLevel(log_level)` to sync log levels (per D-16)
- Log at INFO: `"[OK] Java bridge started on port {port}"`

**Shutdown `stop(self)`:**
- Shutdown gateway, terminate process, wait with timeout
- Log at INFO: `"[OK] Java bridge stopped"`

**Sync pair (private methods):**
- `_sync_to_java(self)`: Push context_manager.get_all() and global_map.get_all() to Java via gateway
- `_sync_from_java(self)`: Pull context and globalMap from Java back to context_manager and global_map
- Both handle None context_manager/global_map gracefully (just skip)

**Every public method that calls Java MUST follow this pattern (per D-12, BRDG-03):**
```
1. _sync_to_java()
2. try: result = self._java_bridge.methodName(...)
3. except Py4JError as e: raise JavaBridgeError(...) from e
4. _sync_from_java()
5. return result
```

| Converter JSON -> Bridge | Schema type strings from converter-generated JSON configs drive type mapping |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | bridge.py _dataframe_to_arrow_bytes | accept | Schema comes from trusted converter-generated JSON configs, not runtime user input. No untrusted data enters the type mapping. |

| T-02-03 | Information Disclosure | bridge.py logging | mitigate | Log data shapes/types but not data values at INFO level. Data values only at DEBUG. ASCII-only output per D-13. |
| T-02-04 | Elevation of Privilege | Groovy expressions | accept | Groovy expressions come from converter-generated JSON configs (trusted). This is a batch ETL system with no user-facing input. Trust boundary is at the converter. |
