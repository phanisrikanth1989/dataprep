    **Shared fixture:**
    ```python
    @pytest.fixture(scope="module")
    def bridge():
        """Start a real Java bridge for the test module."""
        b = JavaBridge()
        b.start()
        yield b
        b.stop()
    ```

    **Test classes:**

    1. `TestTypeRoundTrip` (D-20 -- 12 Talend data types):
       Test round-trip serialization for each of the 12 Talend data types through the bridge. For each type, create a small DataFrame, send it through `execute_java_row` with a pass-through Groovy script (output = input), and verify the data comes back correct.

       - `test_string_round_trip`: DataFrame with str column. Schema: `{"name": "str"}`. Groovy: `output.set("name", input.get("name"))`.
       - `test_integer_round_trip`: DataFrame with Python int. Schema: `{"val": "int"}`. Verify integer values preserved.
       - `test_long_round_trip`: Same as integer (both map to "int" -> int64).
       - `test_float_round_trip`: DataFrame with Python float. Schema: `{"val": "float"}`.
       - `test_double_round_trip`: Same as float (both map to "float" -> float64).
       - `test_big_decimal_round_trip`: DataFrame with Decimal values using DEFAULT precision (38, 18). Schema: `{"amount": "Decimal"}`. Verify precision preserved.
       - `test_big_decimal_custom_precision_round_trip`: **[Addresses review concern: Decimal precision/scale mapping]** DataFrame with `Decimal("123.45")` and `Decimal("9999.99")` values. Schema: `{"amount": "Decimal"}`. Use `extract_precision_map` with schema_columns `[{"name": "amount", "type": "Decimal", "length": 10, "precision": 2}]` to get `precision_map={"amount": (10, 2)}`. Pass this to the bridge call (the bridge's `_df_to_arrow_bytes` should use it when building the Arrow schema). Verify that the returned values have correct scale (2 decimal places). This tests that non-default Decimal precision flows end-to-end from schema config through Python type_mapping through Arrow serialization through Java and back.
       - `test_date_round_trip`: DataFrame with datetime.date. Schema: `{"dt": "datetime"}`.
       - `test_timestamp_round_trip`: DataFrame with datetime.datetime with time component. Schema: `{"ts": "datetime"}`.
       - `test_boolean_round_trip`: DataFrame with bool. Schema: `{"flag": "bool"}`.
       - `test_byte_round_trip`: Small integer values. Schema: `{"val": "int"}` (Byte maps to int).
       - `test_short_round_trip`: Medium integer values. Schema: `{"val": "int"}` (Short maps to int).
       - `test_character_round_trip`: Single-char strings. Schema: `{"ch": "str"}` (Character maps to str).
       - `test_null_values_preserved`: DataFrame with None/NaN values in various typed columns. Verify nulls survive round-trip.
       - `test_multi_column_mixed_types`: DataFrame with str, int, float, bool, datetime columns all at once.

    2. `TestCompiledScripts` (BRDG-06):
       - `test_compile_and_execute`: Compile a tMap script with `compile_tmap_script()`, then execute with `execute_compiled_tmap_chunked()`. Verify output.
       - `test_compiled_script_reuse_across_chunks`: Compile once, call `execute_compiled_tmap_chunked` 3 times with different DataFrames. Verify each call returns correct results (no state leakage between chunks).
       - `test_multiple_compiled_scripts`: Compile 2 different scripts with different component_ids. Execute each. Verify correct results (no cross-script contamination).

    3. `TestBatchExpressions` (BRDG-03 sync):
       - `test_batch_expressions_return_values`: Call `execute_batch_one_time_expressions({"a": "1+1", "b": "'hello'.toUpperCase()"})`. Assert `result["a"] == 2`, `result["b"] == "HELLO"`.
       - `test_globalmap_sync_after_batch`: Set globalMap via bridge, execute expression that reads it, verify value. Then execute expression that modifies globalMap, sync, verify Python-side globalMap updated.
       - `test_context_sync_after_expression`: Set context, execute expression accessing it, verify. Execute expression modifying context, sync, verify Python side updated.

    4. `TestLibraryValidation` (BRDG-04):
       - `test_validate_existing_libraries`: Call `validate_libraries` with path to the bridge JAR itself. Expect success (valid).
       - `test_validate_missing_library`: Call with non-existent path. Expect it in the "missing" result.

    5. `TestPy4JVersion` (BRDG-05):
       - `test_py4j_python_version`: `import py4j; assert py4j.__version__ == "0.10.9.9"` (or similar version check).
       - `test_bridge_connects_successfully`: Verify bridge.start() + stop() works without error (Py4J compatibility confirmed).

    6. `TestLifecycle`:
       - `test_start_stop_cycle`: Start bridge, verify _started=True, stop, verify clean shutdown.
       - `test_double_stop_no_error`: Stop an already-stopped bridge. Should not raise.

    **Important notes:**
    - All tests must clean up (bridge.stop()) even on failure -- use fixtures.
    - Each test that modifies globalMap or context should reset state or use a fresh bridge.
    - Tests should be independent -- order should not matter.
