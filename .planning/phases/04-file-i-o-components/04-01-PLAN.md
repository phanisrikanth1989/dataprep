**_validate_config():** Validate `filepath` is present. Raise `ConfigurationError(f"[{self.id}] Missing required config key 'filepath'")` if missing.

**_process(self, input_data=None) -> dict:** This is a source component (input_data is None). Steps:

1. **Read config values** (per D-04, D-05): Read all config keys using Talend defaults:
   - `filepath` (str), `fieldseparator` (str, default ";"), `row_separator` (str, default "\\n"), `encoding` (str, default "ISO-8859-15"), `header_rows` (int, default 0), `footer_rows` (int, default 0), `limit` (str, default ""), `remove_empty_row` (bool, default True), `csv_option` (bool, default False), `csv_row_separator` (str, default "\\n"), `escape_char` (str, default '"'), `text_enclosure` (str, default '"'), `trim_all` (bool, default False), `trim_select` (list, default []), `check_fields_num` (bool, default False), `check_date` (bool, default False), `die_on_error` (bool, default False)

2. **Warn on deferred features** (per D-21): Check flags `uncompress`, `split_record`, `random`, `advanced_separator`, `enable_decode`. If any is truthy, log `logger.warning(f"[{self.id}] {description} ('{flag}') is not yet implemented. Config flag will be ignored.")`.

3. **Resolve filepath:** Convert via `Path(filepath)`. If file does not exist, raise `FileOperationError(f"[{self.id}] File not found: '{filepath}'")`.

4. **Set pre-execution globalMap variables** (per D-15): Set `{self.id}_FILENAME` = str(resolved filepath), `{self.id}_ENCODING` = encoding. Guard with `if self.global_map:`.

5. **Unescape separators:** Use a `_unescape_separator()` static method that maps `"\\n"` -> `"\n"`, `"\\r\\n"` -> `"\r\n"`, `"\\r"` -> `"\r"`, `"\\t"` -> `"\t"`.

6. **Read file:** Two paths based on `csv_option`:
   - **csv_option=False:** Use `pd.read_csv()` with `sep=field_separator`, `header=None`, `skiprows=header_rows`, `skipfooter=footer_rows`, `encoding=encoding`, `quoting=csv.QUOTE_NONE`, `dtype=str` (read all as strings first for validation), `keep_default_na=False`. If `footer_rows > 0`, must use `engine='python'`. Apply schema column names from `self.output_schema` if available.
   - **csv_option=True:** Use Python `csv.reader()` for RFC4180 compliance. Open file with specified encoding. Create `csv.reader` with `delimiter=field_separator`, `quotechar=text_enclosure`, `escapechar=escape_char if escape_char != text_enclosure else None`, `doublequote=(escape_char == text_enclosure)`. Use `csv_row_separator` (unescaped) for line splitting if different from `"\n"` -- otherwise just iterate lines. Skip `header_rows` lines. Read rows into list. Apply `footer_rows` skip. Build DataFrame from rows with `dtype=str`. Apply schema column names.

7. **Apply limit:** If `limit` is a non-empty string that converts to a positive integer, truncate DataFrame to that many rows: `df = df.head(int(limit))`.

8. **Remove empty rows:** If `remove_empty_row=True`, drop rows where all values are empty string or NaN: `df = df.dropna(how='all')` then drop rows where all cols are empty string.

9. **Row-level validation with REJECT routing** (per D-06, D-07, D-08):
   Initialize `good_rows = []` and `reject_rows = []`. Get expected column count from schema. Iterate through DataFrame rows (use `df.itertuples(index=False)`):

   a. **CHECK_FIELDS_NUM** (FILD-06): If `check_fields_num=True`, check if the number of non-NaN fields matches schema column count. If mismatch, create reject row with `errorCode="FIELD_COUNT"`, `errorMessage=f"Field count mismatch: expected {expected}, got {actual} - Line: {line_num}"`.

   b. **Type conversion** (FILD-03): For each column, attempt type conversion per schema type. Use `_convert_value(value, col_schema)` helper. If conversion fails (ValueError/TypeError), create reject row with `errorCode="TYPE_CONVERSION"`, `errorMessage=f"Cannot convert '{value}' to {target_type} for column '{col_name}' - Line: {line_num}"`.

   c. **CHECK_DATE** (FILD-07): If `check_date=True`, for columns with `date_pattern` in schema, validate using `datetime.strptime(value, pattern)`. If fails, create reject row with `errorCode="DATE_FORMAT"`, `errorMessage=f"Date '{value}' does not match pattern '{pattern}' for column '{col_name}' - Line: {line_num}"`.

   d. Reject rows contain ALL original columns (as strings) plus `errorCode` and `errorMessage` columns.

   e. Good rows get type-converted values.

   Build `main_df = pd.DataFrame(good_rows)` and `reject_df = pd.DataFrame(reject_rows) if reject_rows else None`.

   **Performance note:** For large files without validation flags enabled (`check_fields_num=False`, `check_date=False`), use vectorized pandas type conversion instead of row-by-row. Only fall back to row-by-row when validation/reject routing is needed.

10. **Apply TRIMSELECT / trim_all** (per D-11): If `trim_select` is non-empty, apply per-column trim: for each entry with `trim=True`, strip the string column. Else if `trim_all=True`, strip all object columns. Apply BEFORE type conversion for the vectorized path, AFTER reading for the row-by-row path.

11. **Schema validation on good rows:** If `self.output_schema` exists, call `self.validate_schema(main_df, self.output_schema)` on the good rows only. This handles final type coercion.

12. **Return result:** `return {"main": main_df, "reject": reject_df}`. BaseComponent auto-updates NB_LINE, NB_LINE_OK, NB_LINE_REJECT stats.

**Static helpers to implement:**
- `_unescape_separator(sep: str) -> str` -- convert escaped separators
- `_convert_value(value: str, col_schema: dict) -> Any` -- convert string value per schema type (str, int, float, bool, datetime, Decimal)
- `_validate_date(value: str, pattern: str) -> bool` -- check date against pattern

**CRITICAL per D-04:** NEVER read config key `delimiter` -- only `fieldseparator`. NEVER read `remove_empty_rows` (plural) -- only `remove_empty_row` (singular).
