# Phase 07.2 Summary: validate-config bug sweep

## What Was Done

Eliminated the systemic bug class where `_validate_config()` (BaseComponent.execute()
Step 2) inspected the *content* of config fields BEFORE context resolution (Step 3),
causing spurious rejection or crashes when fields held `${context.X}` references.

Across the 11 in-scope components:

- 5 Group A confirmed bugs were fixed by moving content checks (int coercion,
  length checks, `.isdigit()`, range checks) from `_validate_config()` into
  `_process()` -- after self.config has been fully resolved.
- 5 Group B suspect sites were resolved with verdicts: 4 KEEP (closed-list enums
  or fields the converter never emits as context vars) + 1 DEFER fix
  (send_mail.smtp_port).
- 1 process rule (Rule 12) was added to `MANUAL_COMPONENT_AUTHORING.md` so future
  contributors cannot reintroduce the pattern.

## Process Rule Added (Rule 12)

Plan 03 adds Rule 12 to `docs/v1/standards/MANUAL_COMPONENT_AUTHORING.md`:

> "Rule 12: `_validate_config` may only check key presence and container shape --
> never content."

Allowed in `_validate_config`: key presence, `isinstance` shape, container shape.
Disallowed: `len()`, regex, `int()`/`float()` coercion, enum membership for
non-closed-lists, `os.path.exists`, numeric range. Disallowed checks belong in
`_process()` after Step 3 resolution.

Heading updated from "11 Rules" to "12 Rules". Cross-references
`base_component.py:204-260` execute lifecycle and the Phase 07.2 sweep.

## Patterns Established

- **Deferred-check pattern:** `_validate_config` docstring note + `_process`
  check with same exception type (`ConfigurationError`) and identical message
  string. Documented as Rule 12 in `MANUAL_COMPONENT_AUTHORING.md`.
- **Group B KEEP rationale comment template:**
  `"Group B verdict (Phase 07.2): KEEP. <field> is <emitted-as-int|closed-list|never-extracted>
  by the converter at <converter file:line>; <isinstance | membership> check is
  shape-only, allowed by Rule 12."`
- **DEFER fix exception placement (send_mail):** new content check placed OUTSIDE
  the broad `try/except Exception` block so `ConfigurationError` surfaces
  directly to the engine rather than being re-wrapped as
  `ComponentExecutionError`. Mirrors the pre-existing `_validate_config` error
  path.
- **Test fixture pattern for direct `_validate_config` / `_process` calls:**
  manually populate `comp.config = dict(config)` before calling these methods
  in isolation, since `BaseComponent.__init__` only sets `_original_config` and
  `self.config = {}` -- only `execute()` Step 1 deepcopies `_original_config`
  into `self.config`. Documented in test docstrings of all new test files.

## Decisions Made

- Group A scope locked to the 5 components and 9 fields enumerated in CONTEXT.md
  decision A. `ARCHIVE_FORMAT` and `file_input_excel` `footer` / `limit` were
  intentionally retained as out-of-scope.
- All raised errors in `_process` use `ConfigurationError` (matching
  `exceptions.py` hierarchy) and preserve the original message text.
- `context_load.py` KEEP comment lists all 4 valid policies
  (`WARNING` / `ERROR` / `INFO` / `NO_WARNING`) -- corrected against the actual
  `_VALID_POLICIES` constant during execution (the plan listed only 3).
- `send_mail.smtp_port` resolution accepts both `int` and `string-of-int` (e.g.
  resolved `"${context.SMTP_PORT}"` -> `"587"`).

## Lessons Carried Forward

- Phase 8 (Code Components) onward must reference Rule 12 when authoring new
  components. Rule 12 is the durable artifact preventing recurrence.
- Future audits of pre-resolution content checks should be unnecessary now that
  the rule is documented and pinned by tests.
- Out-of-scope items (`ARCHIVE_FORMAT`, `file_input_excel` `footer`/`limit`)
  remain candidates for a follow-up sweep if a Talend job is found using
  context vars for those fields.
- BaseComponent lifecycle is non-obvious for direct unit testing of
  `_validate_config` and `_process`. The "manually deepcopy `_original_config`
  into `self.config` in the test fixture" pattern is now established and
  documented in the new test files.
