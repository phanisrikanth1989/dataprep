## Decisions Made

- **send_mail ConfigurationError placement:** Original code at line 145 raised ConfigurationError directly. Inserting the new smtp_port check INSIDE the existing `try:` block at line 147 would have it caught by the broad `except Exception` re-wrapping it as ComponentExecutionError -- a behavior change. Resolution: place the new smtp_port check OUTSIDE the broad try/except, mirroring the pre-existing error path. Preserves original exception type and message.
- **context_load enum list correction:** The plan's KEEP comment example listed `WARNING / ERROR / NO_WARNING` (3 values). Actual `_VALID_POLICIES` constant in context_load.py contains `{"ERROR", "WARNING", "INFO", "NO_WARNING"}` (4 values). Comment expanded to all 4 to keep the rationale accurate. Logged as deviation Rule 1 (bug -- inaccurate doc would mislead future contributors).
- **Test fixture pattern:** New tests for fixed_flow_input and send_mail mirror the BaseComponent execute() Step 1 deepcopy by manually setting `comp.config = dict(config)` before calling `_validate_config()` / `_process()`. Without this, `self.config` is empty (only `_original_config` is set on `__init__`).
