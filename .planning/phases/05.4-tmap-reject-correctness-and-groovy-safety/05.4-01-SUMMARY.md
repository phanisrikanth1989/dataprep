## Decisions Made

- **Partial extraction for filter path.** Per the plan's Task 2 action and PATTERNS.md, the inline per-row loop in `_evaluate_outputs_py` is kept for outputs with `activate_filter=True and filter != ""` because the kept/rejected row split must happen in lockstep. Plan 05.4-03 will lift the filter path onto `_evaluate_output_columns_py` via a precomputed keep-mask. This is a deliberate two-step refactor, not a scope cut: the helper API is stable now so Plans 02-04 can build against it.
- **Empty fast path.** When `row_source.empty`, the helper returns `pd.DataFrame(columns=col_names)` without iterating. This honours R7 single-allocation and matches the existing terminal allocation pattern at `map.py:2463-2466`.
- **`_eval_output_row` unchanged.** The new helper is a pure loop wrapper -- no new eval semantics. `_VarBag`, `_build_namespace`, `_build_row_dicts_for_py` are reused as-is from Phase 05.3.
