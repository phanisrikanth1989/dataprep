2. **File Structure** -- Complete code template showing:
   - Module docstring with config mapping header (same style as converter pattern)
   - Imports (from ...base_component import BaseComponent, from ...exceptions import ...)
   - Class definition extending BaseComponent
   - `__init__` calling super().__init__()
   - `_validate_config()` implementation (check required config keys, raise ConfigurationError)
   - `_process()` implementation (the actual work -- return {'main': df, 'reject': reject_df})
   - Static helper methods (prefixed with _)
   - No print() statements -- use logger

3. **Numbered Rules (MANDATORY):**
   - Rule 1: Component class extends `BaseComponent` (or `BaseIterateComponent` for iterate)
   - Rule 2: `_validate_config()` is required -- check ALL config keys component depends on. Raise `ConfigurationError` with descriptive message.
   - Rule 3: `_process()` MUST return dict with `'main'` key (pd.DataFrame or None). Optional: `'reject'` key for rejected rows.
   - Rule 4: NEVER override `execute()`. The lifecycle in BaseComponent handles validation, config resolution, mode selection, stats, and error handling. Override only `_validate_config()` and `_process()`.
   - Rule 5: Read config values via `self.config.get('key', default)` inside `_process()`, never in `__init__()`. Config is resolved fresh each execute().
   - Rule 6: Use `self.global_map.put()` and `self.global_map.get()` for inter-component communication. Stats are handled by BaseComponent automatically.
   - Rule 7: Use custom exceptions: `ConfigurationError` for bad config, `DataValidationError` for bad data, `FileOperationError` for file I/O failures. Never raise generic `Exception` or `RuntimeError`.
   - Rule 8: Use `logger = logging.getLogger(__name__)` at module level. Log at INFO for lifecycle events, DEBUG for data details, WARNING for degraded operation, ERROR for failures. Never use print().
   - Rule 9: Register component in `COMPONENT_REGISTRY` in `engine.py` with both V1 name and Talend alias.
   - Rule 10: Component MUST work after `reset()` -- no state leaks between iterate re-executions. All mutable state initialized in `_process()`, not stored on `self` across calls.

4. **Config Access Pattern:**
Show the correct way to read config, emphasizing that config is resolved by BaseComponent before _process() is called.

5. **REJECT Flow Pattern:**
Show how to route rows to reject output:
```python
def _process(self, input_data=None) -> dict:
    good_rows = input_data[mask]
    bad_rows = input_data[~mask]
    return {
        'main': good_rows,
        'reject': bad_rows if not bad_rows.empty else None,
    }
```

6. **GlobalMap Variables Pattern:**
Show how to set component-specific globalMap variables (e.g., {id}_FILENAME):
```python
def _process(self, input_data=None) -> dict:
    # ... processing ...
    if self.global_map:
        self.global_map.put(f"{self.id}_FILENAME", file_path)
    return {'main': df, 'reject': None}
```

7. **Anti-Patterns Section** (matching CONVERTER_PATTERN.md style):
   - Do NOT override execute()
   - Do NOT read config in __init__
   - Do NOT mutate self._original_config
   - Do NOT use print()
   - Do NOT raise generic Exception
   - Do NOT store processing state on self (breaks iterate)

Brief section showing how to extend BaseIterateComponent instead of BaseComponent for iterate-producing components.
