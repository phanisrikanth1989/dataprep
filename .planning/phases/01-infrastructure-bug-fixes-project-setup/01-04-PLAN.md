Rewrite `src/v1/engine/trigger_manager.py` from scratch. The new implementation must:

1. **Preserve TriggerType and Trigger classes** (they are correct). Keep the same enum values and Trigger constructor signature.

2. **Fix ENG-06 -- Operator replacement order**: In `_evaluate_condition()`, replace multi-character operators BEFORE single-character ones:
   - Step 1: Replace `!=` with a placeholder token (e.g., `__NEQ__`)
   - Step 2: Replace `!` with ` not ` (for boolean negation)
   - Step 3: Restore `__NEQ__` back to `!=`
   - OR use regex: `re.sub(r'!(?!=)', ' not ', condition)` which uses negative lookahead to skip `!=`
   - Also handle: `&&` -> `and`, `||` -> `or`, `null` -> `None`, `true` -> `True`, `false` -> `False`

3. **Fix NEW-04 -- Sandboxed eval**: Use restricted globals:
```python
_SAFE_GLOBALS = {"__builtins__": {}, "None": None, "True": True, "False": False, "int": int, "str": str, "float": float, "bool": bool}
result = eval(python_condition, _SAFE_GLOBALS, local_vars)
```
Catch `Exception` and raise `TriggerEvaluationError` with the original condition string for debuggability.

4. **Fix NEW-05 -- All cast types**: Parse ALL Java cast patterns, not just `((Integer)...)`:
```python
# Pattern: ((Type)globalMap.get("key"))
_CAST_PATTERN = re.compile(r'\(\((\w+)\)globalMap\.get\("([^"]+)"\)\)')
```
Support cast types: Integer, Long, Short, Byte -> `int(value)`, Float, Double -> `float(value)`, Boolean -> `bool(value)`, String -> `str(value)`. Map each cast type to the appropriate Python converter.

5. **Fix ENG-10 -- OnSubjobOk timing**: `should_fire_trigger(trigger)` for OnSubjobOk must check that ALL components in the source subjob have status "ok" or "success", not just the trigger's from_component. Use `self.subjob_components` and `self.component_to_subjob` mappings:
```python
if trigger.type == TriggerType.ON_SUBJOB_OK:
    subjob_id = self.component_to_subjob.get(trigger.from_component)
    if subjob_id:
        components = self.subjob_components.get(subjob_id, [])
        return all(self.component_status.get(c) in ('ok', 'success') for c in components)
```

6. **Methods to preserve** (reimplemented cleanly):
   - `add_trigger(self, trigger_type, from_component, to_component, condition=None)`
   - `register_subjob(self, subjob_id, component_ids)` -- track subjob membership

   - `_evaluate_condition(self, condition)` -- safe condition evaluation
