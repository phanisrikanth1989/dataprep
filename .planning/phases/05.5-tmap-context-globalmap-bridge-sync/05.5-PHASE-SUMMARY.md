## Key decisions

- **Direct dict-write bypasses `set_context` str-coercion** (SPEC L127 / RESEARCH Pitfall 1) -- the setter source is unchanged in this phase; sync sites write `bridge.context[k] = v` directly to preserve type fidelity
- **Per-key writes preserve `__rejectMode__` flag** (SPEC L128 / RESEARCH Pitfall 2) -- never `bridge.context.clear()` or `bridge.context = dict(...)`
- **No shared `BaseComponent` utility** -- each component owns its own `_push_runtime_state_to_bridge` helper (per SPEC, keeps the surface minimal and lets each component decide invocation timing)
- **PyMap uses a different mechanism** (pure Python eval-namespace binding via `_ContextView` / `_GlobalMapView`) since it doesn't go through the Java bridge -- R5 compliant
- **R9 path (b)** chosen -- 2 catch xfails promoted (gap closed by 05.5-02 + 05.5-07), 4 filter-reject xfails retained (orthogonal scope deferred to Phase 05.6)
- **JAR is gitignored on this branch** (Phase 13 Plan 01 convention) -- Java source in `6c3c36f` is the durable source-of-truth; runtime JAR rebuilt on disk via `mvn package`
