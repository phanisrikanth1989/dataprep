## Decisions Made

- **Wrapper view over `types.SimpleNamespace` snapshot:** Used live-view wrappers that delegate to `cm.get_all()` / `gm.get(...)` per access rather than snapshotting the values into a SimpleNamespace at namespace-build time. Reason: SimpleNamespace would freeze state at the moment `_build_namespace` is called; the view keeps semantics consistent with how `globalMap.get` works in tMap's Groovy binding (reads current Java-side state every call).
- **Missing-key behavior diverges by access form:** Attribute access raises `AttributeError`; item access raises `KeyError`. This matches Python conventions and gives expression authors a deterministic, debuggable failure rather than a silent `None` that would propagate downstream as `TypeError` deep in arithmetic.
- **None-handling:** When `context_manager` or `global_map` is `None`, the views still get installed but each access raises `AttributeError`/`KeyError` deterministically. Avoids the alternative of conditionally omitting the keys, which would produce a confusing `NameError` from `eval()`.
