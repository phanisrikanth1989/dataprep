## Critical Anti-Patterns (do NOT repeat these)

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Manual `self.validate_schema()` in `_process()` | Bypasses BaseComponent's lifecycle; causes double-validation, ordering bugs, die_on_error reject-routing bypass. Rule 11 — most-violated contract pre-7.1. | advisory | When authoring a Phase 8 component, do NOT call `self.validate_schema()`. BaseComponent step 7c does it automatically. The new MANUAL_COMPONENT_AUTHORING.md spells this out. |
| Calling `_update_stats()` inside `_process()` | Causes double-counting because BaseComponent calls `_update_stats_from_result` after. Pre-7.1 tests were calibrated to this bug. | advisory | Subclasses MUST NOT touch stats. Return `{"main": ..., "reject": ...}` and let BaseComponent count. |
| Manager-style commits outside GSD workflow | The 48-finding audit (Phase 7.1's reason for existing) was triggered by manager commits introducing regressions across already-shipped phases. | advisory | If a contributor doesn't use GSD, they MUST follow MANUAL_COMPONENT_AUTHORING.md and the PR checklist at the bottom of that doc. |
