# Example enrichment-requirements document

`sample_enrichment_requirements.docx` is a ready-to-run test document for the
multi-agent enrichment system. It is authored in the exact format
`extract_doc` requires (see `agents/templates/etl_requirements_template.md`)
and passes the conformance gate.

## The scenario

A daily **settlements feed** (`transactions`) is enriched against two reference
sources before it goes to SmartStream TLM:

- **R1 (join)** add `counterparty_name` + `country` from `counterparties` on `counterparty_code`
- **R2 (join)** add `currency_name` from `currencies` on `currency_code`
- **R3 (sort)** order the result by `txn_id`

Both joins are LEFT joins, so every transaction is kept. The sample data is
built to exercise the interesting cases:

- `T004` uses `CP999` -> **no counterparty match** -> `counterparty_name`/`country` stay empty
- `T005` uses `JPY` -> **no currency match** -> `currency_name` stays empty
- `T003` has a **null** `narrative`

The `Expected Output` table is the deterministic result of those rules and is
what the harness diffs the engine output against (the oracle). `txn_id*` is
marked as the composite key.

## How to use it

Validate/extract it standalone (no Copilot needed):

```bash
python -m agents.tools.extract_doc agents/examples/sample_enrichment_requirements.docx --out extracted.json
# exit 0 = conformant; extracted.json holds schema + rules + sample + expected + derived facts
```

Or drop it into the VS Code Copilot enrichment workflow and let the
orchestrator drive the six specialists end-to-end (extract -> interpret ->
design -> configure -> assemble -> run -> diff -> human gate).

## Extending it

Add sources, rules, or outputs by editing the four Heading-1 blocks - but note
the **`Expected Output` rows are the test oracle**, so they must equal the
correct result of your rules or the harness will (correctly) report a mismatch.
If you add a `schema_validate` cast (e.g. `amount` -> `decimal`), remember the
engine strips trailing zeros (`"10.50"` -> `"10.5"`), so the expected values
must reflect the cast output. The `.docx` was generated by
`scratchpad/gen_docx.py`-style python-docx code; regenerate rather than
hand-editing tables if you change the schema substantially.
